# 课程介绍与学习指南

> 欢迎来到 JavaWeb + AI 实训课程！
> 本文档将带你全面了解这门课程的目标、技术栈、学习路线以及你需要准备的一切。
> 如果你是零基础学员，请耐心读完；如果你有编程基础，可以快速浏览技术栈和版本差异部分。

## 一、学习目标

完成本课程后，你将具备以下能力：

- **独立搭建企业级 Web 开发环境**：从零配置 JDK、Maven、Node.js、MySQL、Redis、Docker 等全套工具链，不再依赖他人帮忙配环境。
- **掌握前后端分离开发模式**：理解前端 Vue 与后端 SpringBoot 如何协作，能够独立完成接口联调。
- **完成一个完整的 CRUD 人事管理系统（Tlias）**：包括部门管理、员工管理、登录认证、文件上传、分页查询等核心功能。
- **理解企业级开发规范**：掌握 RESTful API 设计、Git 分支管理、代码提交规范、分层架构设计等团队协作必备技能。
- **（版本 B 专属）具备 AI 功能集成能力**：能够将大语言模型能力接入 Web 应用，实现 NL2SQL、RAG 知识库、AI 简历解析等功能。
- **具备项目部署与运维基础能力**：使用 Docker 打包应用、配置 Nginx 反向代理、实现一键启动完整技术栈。

## 二、核心知识点

### 2.1 这门课程是什么？为什么值得你学？

**是什么**：这是一套面向零基础学员的 JavaWeb 全栈实训课程，采用"双版本并行"设计——版本 A 夯实基础，版本 B 进阶实战并融入 AI 能力。

**为什么需要它**：
- 传统教学往往"重理论轻实践"，学完后面对真实项目依然无从下手。
- 网上教程碎片化严重，前端、后端、数据库各自为政，缺乏一个完整的项目串联。
- AI 时代已经到来，只会传统 CRUD 已不够，需要了解如何将 AI 能力集成到业务系统中。
- 本课程用一个真实的"Tlias 人事管理系统"贯穿始终，每学一个知识点都能立刻看到效果。

**真实场景类比**：
> 想象你要学做菜。传统方式是先背 100 种食材的特性，再背 50 种刀法，最后才让你炒一盘菜——很多人中途就放弃了。
> 本课程的方式是：第一天就教你做一道"番茄炒蛋"（写一个页面），第二天教你做"红烧肉"（写一个接口），
> 20 天后你已经能独立完成一桌"年夜饭"（完整的人事管理系统）。

### 2.3 A/B 版本差异对照表

本课程提供两个版本，你可以根据自己的基础和目标选择：

| 对比维度 | 版本 A（基础实训版） | 版本 B（AI 增强实战版） |
|---------|-------------------|----------------------|
| **适合人群** | 完全零基础，第一次接触 Web 开发 | 有一定基础，希望产出简历级项目 |
| **SpringBoot** | 2.7.18 | 3.2.x |
| **JDK 版本** | JDK 8 | JDK 17 |
| **持久层框架** | MyBatis（XML 配置） | MyBatis-Plus 3.5.5（注解为主） |
| **认证方案** | JWT + 自定义 Filter | Sa-Token + Redis 会话 |
| **前端框架** | Vue 2.6 | Vue 3.5（Composition API） |
| **UI 组件库** | ElementUI | Element Plus |
| **缓存** | 无 | Redis 7 |
| **AI 能力** | 无 | Spring AI + 大模型 API |
| **项目复杂度** | 标准 CRUD + 登录 | CRUD + 登录 + AI 功能模块 |
| **简历价值** | 证明你有基础开发能力 | 证明你能做 AI 全栈项目 |

**版本选择建议**：

```
你是零基础学员？
    |
    +-- 是 --> 先学版本 A，完成后再看版本 B 的差异部分
    |
    +-- 否 --> 直接学版本 B，跳过你已掌握的基础章节
```

### 2.8 学习建议：如何高效学习本课程

#### 建议一：先理解再记忆，不要死记硬背

编程不是背课文。遇到新概念时：
1. 先问"这个东西解决什么问题？"
2. 再看"它是怎么解决的？"
3. 最后"我自己能不能写出来？"

#### 建议二：代码一定要亲手敲

看十遍不如敲一遍。即使文档里给了完整的代码，也请自己打开 IDE 亲手输入。
敲代码的过程中你会：
- 熟悉 IDE 的自动补全和快捷键
- 发现拼写错误（这是最常见的 bug 来源）
- 加深对语法结构的理解

#### 建议三：善用错误信息

遇到报错不要慌，按以下步骤排查：

```
看到报错信息
    |
    +-- 第 1 步：看错误类型（Error 还是 Warning？）
    |
    +-- 第 2 步：看错误位置（哪个文件？第几行？）
    |
    +-- 第 3 步：看错误描述（关键提示词是什么？）
    |
    +-- 第 4 步：复制错误信息到搜索引擎查找
    |
    +-- 第 5 步：对照本文档的"常见错误排查"表格
    |
    +-- 第 6 步：如果以上都无效，再寻求帮助
```

#### 建议四：如何提问

当需要向老师或同学求助时，请提供以下信息：

1. **你正在做什么？**（"我在尝试启动 SpringBoot 项目"）
2. **你期望发生什么？**（"期望看到 Tomcat 启动成功的日志"）
3. **实际发生了什么？**（"报错：Failed to configure a DataSource"）
4. **你已经尝试了哪些方法？**（"我检查了 application.yml 里的数据库配置"）
5. **相关代码/配置**（贴出关键代码片段，不要贴整个文件）
6. **完整的错误堆栈**（从控制台复制，不要只贴最后一行）

#### 建议五：建立自己的知识库

建议每天学习结束后整理笔记，推荐格式：

### 练习 3：启动第一个 Docker 容器

```bash
# 步骤 1：确保 Docker Desktop 正在运行
# 在系统托盘找到 Docker 图标，确认状态为"Running"

# 步骤 2：启动 MySQL 容器
docker run -d \
  --name tlias-mysql-test \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -e MYSQL_DATABASE=tlias_db \
  mysql:8.0

# 参数说明：
# -d          后台运行（detached 模式）
# --name      给容器起个名字，方便后续管理
# -p 3306:3306  将主机的 3306 端口映射到容器的 3306 端口
# -e           设置环境变量，这里设置 root 密码和默认数据库名
# mysql:8.0   使用的镜像名称和版本

# 步骤 3：查看容器是否启动成功
docker ps
# 预期看到 tlias-mysql-test 容器在运行中

# 步骤 4：用 Navicat 或 DBeaver 连接测试
# 主机：localhost
# 端口：3306
# 用户名：root
# 密码：123456
# 如果能连接成功，说明 MySQL 容器正常工作

# 步骤 5：停止并删除测试容器
docker stop tlias-mysql-test
docker rm tlias-mysql-test
```

**完成标准**：能用数据库工具成功连接到 Docker 中的 MySQL。

## 四、常见错误排查

| 阶段 | 错误现象 | 可能原因 | 解决方案 |
|------|---------|---------|---------|
| **环境安装** | `java 不是内部或外部命令` | JDK 未安装或环境变量未配置 | 重新安装 JDK，并检查系统 Path 变量是否包含 JDK 的 bin 目录 |
| **环境安装** | `mvn 不是内部或外部命令` | Maven 未安装或环境变量未配置 | 安装 Maven 并将 bin 目录加入 Path，或直接使用 IDEA 内置 Maven |
| **环境安装** | `node 不是内部或外部命令` | Node.js 未安装 | 从官网下载安装包重新安装，安装时勾选"Add to PATH" |
| **项目导入** | IDEA 中 pom.xml 报红 | Maven 依赖下载失败 | 检查网络（能否访问 maven.aliyun.com）；在 IDEA 中右键 pom.xml → Maven → Reimport |
| **项目导入** | `npm install` 报错 | 网络问题或 npm 源问题 | 切换淘宝镜像：`npm config set registry https://registry.npmmirror.com` |
| **项目导入** | `npm install` 卡住不动 | 网络超时 | 按 Ctrl+C 取消，重新执行；或使用手机热点重试 |
| **Docker 启动** | `docker: Cannot connect to the Docker daemon` | Docker Desktop 未启动 | 打开 Docker Desktop，等待状态变为 Running |
| **Docker 启动** | `port is already allocated` | 本地已有服务占用了该端口 | 停止本地 MySQL 服务，或修改 Docker 端口映射（如 -p 3307:3306） |
| **Docker 启动** | 容器启动后立即退出 | 容器内应用启动失败 | 查看日志：`docker logs 容器名`，根据错误信息排查 |
| **版本检查** | JDK 版本不对（需要 8 却装了 17） | 安装了多个 JDK 版本 | 检查 JAVA_HOME 环境变量指向的 JDK 路径；或在 IDEA 中单独配置项目 JDK |
