# AI 简历解析与自动入库

## 学习目标

- 掌握 PDF 文本提取技术
- 使用大模型实现结构化信息抽取
- 实现简历解析后的人工确认交互

## 核心知识点

### 1. PDF 文本提取

```java
import org.apache.pdfbox.pdmodel.PDDocument;
import org.apache.pdfbox.text.PDFTextStripper;

try (PDDocument document = PDDocument.load(inputStream)) {
    PDFTextStripper stripper = new PDFTextStripper();
    stripper.setSortByPosition(true); // 保留布局信息
    String text = stripper.getText(document);
}
```

**注意事项**：
- 扫描版 PDF（图片）无法直接提取文本，需 OCR 预处理
- 表格布局可能导致文本顺序混乱
- 中英文混排需要特殊处理

### 2. 结构化输出

使用 Spring AI 的 `BeanOutputConverter` 强制 LLM 返回 JSON：

```java
public record ResumeParseResult(
    String name,
    String phone,
    String email,
    List<Education> education,
    List<WorkExperience> workExperience,
    List<String> skills
) {}

BeanOutputConverter<ResumeParseResult> converter =
    new BeanOutputConverter<>(ResumeParseResult.class);

String response = chatClient.prompt()
    .system("从简历中提取结构化信息，严格按JSON格式输出")
    .user(resumeText)
    .call()
    .content();

ResumeParseResult result = converter.convert(response);
```

### 3. 前端交互设计

1. **拖拽上传区域**：支持 PDF 文件拖拽上传
2. **解析进度条**：显示 AI 解析进度
3. **结构化预览卡片**：分块展示提取的字段
4. **人工确认**：低置信度字段标红，允许用户修改
5. **一键入库**：确认后自动填充员工档案表单

### 4. 数据映射

```
简历解析结果 → 员工档案
name       → emp.name
phone      → emp.phone
email      → emp.email
skills     → 技能标签（可扩展字段）
education  → 学历信息
workExperience → 工作经历（emp_expr表）
```

## 常见错误排查

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| PDF解析为空 | 扫描版PDF | 提示用户上传文本版PDF |
| 字段提取不全 | 简历格式特殊 | 优化Prompt，增加示例 |
| JSON解析失败 | LLM输出格式不规范 | 使用BeanOutputConverter自动校验 |

## 本节小结

AI简历解析的核心价值是"将非结构化文本转化为结构化数据"。技术难点不在解析本身，而在于处理各种简历格式的边界情况，以及设计好"AI解析 + 人工确认"的交互流程。
