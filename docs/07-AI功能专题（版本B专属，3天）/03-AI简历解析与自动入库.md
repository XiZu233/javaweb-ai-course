# AI 简历解析与自动入库

## 学习目标

- 理解 AI 简历解析的业务价值：将非结构化简历转化为可检索的结构化数据，解决 HR 手动录入效率低、易出错的痛点
- 掌握 PDF 文本提取技术（Apache PDFBox），能够处理文本版 PDF 的读取与布局保留
- 熟练使用 Spring AI 的 `BeanOutputConverter` 实现 LLM 结构化输出，将大模型返回的 JSON 自动映射为 Java 对象
- 掌握 Prompt 工程技巧（System Prompt 角色定义、Few-shot 示例、字段置信度评分），提升解析准确率
- 掌握简历解析结果的校验、人工确认交互设计，以及最终的数据库入库流程（含事务控制）
- 能够独立完成从前端拖拽上传到后端解析再到入库的完整链路开发

## 核心知识点

### 1. 为什么需要 AI 简历解析

#### 1.1 HR 手动录入的痛苦

想象一下，一家中型公司每天收到 50 份简历，HR 需要：

1. 打开每份 PDF/Word 简历
2. 逐行阅读，找到姓名、电话、邮箱、学历、工作经历
3. 切换到员工管理系统，手动复制粘贴到对应字段
4. 一份简历平均耗时 3-5 分钟，一天就是 3-4 小时
5. 复制粘贴容易出错：电话号码少一位、邮箱域名写错、时间线搞混

**核心痛点总结**：

| 痛点 | 具体表现 | 成本 |
|------|---------|------|
| 时间成本高 | 每份简历 3-5 分钟，批量处理耗时巨大 | 人力成本 |
| 错误率高 | 手动录入难免出错，影响后续联系 | 沟通成本 |
| 格式不统一 | 简历模板五花八门，没有固定格式 | 适配成本 |
| 信息检索难 | 非结构化文本无法按条件筛选 | 管理成本 |
| 历史简历沉睡 | 过去收到的简历无法有效复用 | 机会成本 |

#### 1.2 AI 解析的价值

AI 简历解析的核心价值可以用一句话概括：**将非结构化文本转化为结构化数据**。

```
+------------------+     +------------------+     +------------------+
|   非结构化简历    | --> |   AI 结构化抽取   | --> |   结构化数据库    |
|  (PDF/Word/图片) |     |  (LLM + Schema)  |     |  (MySQL 表)      |
+------------------+     +------------------+     +------------------+
       人工 5 分钟              自动 5 秒                 可检索、可筛选
```

解析后的结构化数据可以：
- 按技能标签筛选候选人（如"会 Vue3 + SpringBoot"）
- 按工作年限排序
- 按学历层次过滤
- 全文检索关键词

### 2. 简历解析的技术链路

完整的简历解析与入库流程如下：

```
+----------------+    +----------------+    +----------------+    +----------------+
|  1. 文件上传    | -> |  2. 文本提取    | -> |  3. AI 结构化   | -> |  4. JSON 校验   |
|  PDF/Word/图片  |    |  PDFBox / OCR  |    |   LLM 抽取     |    | BeanOutputConverter |
+----------------+    +----------------+    +----------------+    +----------------+
                                                                         |
                                                                         v
+----------------+    +----------------+    +----------------+    +----------------+
|  8. 完成入库    | <- |  7. 事务提交    | <- |  6. 人工确认    | <- |  5. 置信度评估   |
|  返回成功提示   |    |  @Transactional |    |  前端编辑修正   |    |  低置信度标红   |
+----------------+    +----------------+    +----------------+    +----------------+
```

**各阶段说明**：

| 阶段 | 处理内容 | 关键组件 |
|------|---------|---------|
| 1. 文件上传 | 接收前端上传的简历文件，校验格式和大小 | ResumeController |
| 2. 文本提取 | 从 PDF/Word 中提取纯文本，保留段落结构 | PDFBox / POI |
| 3. AI 结构化 | 将简历文本发送给 LLM，按 Schema 抽取字段 | Spring AI + Kimi |
| 4. JSON 校验 | 将 LLM 返回的 JSON 转换为 Java 对象 | BeanOutputConverter |
| 5. 置信度评估 | 检查各字段的置信度分数，标记低置信度项 | 业务逻辑 |
| 6. 人工确认 | 前端展示解析结果，允许用户修改 | Vue3 + ElementPlus |
| 7. 事务提交 | 将确认后的数据写入数据库，保证一致性 | @Transactional |
| 8. 完成入库 | 返回操作结果，更新前端状态 | REST API |

### 3. PDF 文本提取详解

#### 3.1 Apache PDFBox（文本版 PDF）

PDFBox 是 Apache 开源的 Java PDF 库，可以读取、创建、修改 PDF 文档。对于简历解析，我们主要使用它的文本提取功能。

**为什么选 PDFBox？**

| 库 | 优点 | 缺点 | 适用场景 |
|----|------|------|---------|
| Apache PDFBox | 开源免费、功能全面、社区活跃 | 对复杂表格支持一般 | 文本版 PDF |
| iText | 功能强大、表格处理较好 | 开源版本功能受限 | 商业项目 |
| PDF.js | 浏览器端解析 | 需要前端环境 | 纯前端方案 |
| Tesseract | 支持 OCR | 只处理图片 | 扫描版 PDF |

**基础文本提取代码**：

```java
// ============================================
// ResumePdfExtractor.java - PDF 文本提取工具类
// ============================================

import org.apache.pdfbox.pdmodel.PDDocument;           // PDF 文档对象，代表整个 PDF 文件
import org.apache.pdfbox.text.PDFTextStripper;         // 文本提取器，负责从 PDF 中抽取文字
import org.springframework.stereotype.Component;        // Spring 组件注解，标记为可注入的 Bean
import org.springframework.web.multipart.MultipartFile; // Spring 文件上传对象

import java.io.InputStream; // 字节输入流，用于读取上传的文件内容

@Component // 标记为 Spring 组件，可以在 Service 中通过 @Autowired 注入使用
public class ResumePdfExtractor {

    /**
     * 从上传的 PDF 文件中提取纯文本
     *
     * @param file 前端上传的 PDF 文件（MultipartFile 是 Spring 封装的上传文件对象）
     * @return 提取出的纯文本字符串，段落之间用换行符分隔
     */
    public String extractText(MultipartFile file) {
        // try-with-resources 语法：自动关闭资源，避免内存泄漏
        // PDDocument.load() 将上传文件的字节流解析为 PDF 文档对象
        try (InputStream inputStream = file.getInputStream();
             PDDocument document = PDDocument.load(inputStream)) {

            // 创建文本提取器实例
            PDFTextStripper stripper = new PDFTextStripper();

            // setSortByPosition(true) 按页面位置排序文本
            // 作用：保留简历的段落顺序，避免文字乱序
            // 例如：左侧的"姓名"和右侧的"张三"会按正确顺序输出
            stripper.setSortByPosition(true);

            // setAddMoreFormatting(true) 保留更多格式信息
            // 作用：在文本中保留换行和空格，帮助区分不同段落
            stripper.setAddMoreFormatting(true);

            // getText() 执行实际的文本提取，遍历所有页面并拼接文本
            String text = stripper.getText(document);

            // 返回提取的文本，如果 PDF 是扫描版（图片），这里会返回空字符串
            return text;

        } catch (Exception e) {
            // 提取失败时抛出运行时异常，由上层统一处理
            // 例如：文件损坏、不是 PDF 格式、PDF 被加密等情况
            throw new RuntimeException("PDF 文本提取失败: " + e.getMessage(), e);
        }
    }
}
```

#### 3.2 扫描版 PDF 的 OCR 方案

**什么是扫描版 PDF？**

扫描版 PDF 本质上是图片集合。HR 收到的简历可能是：
- 纸质简历用扫描仪扫成的 PDF（每一页都是一张图片）
- 手机拍照后合成的 PDF
- 截图保存的 PDF

这类 PDF 用 PDFBox 提取文本会得到空字符串，因为里面没有文本层，只有图片。

**OCR 方案对比**：

```
+-------------------+     +-------------------+     +-------------------+
|   扫描版 PDF      | --> |    OCR 识别       | --> |    纯文本         |
|   (图片集合)      |     |  (文字识别引擎)    |     |  (可提取信息)      |
+-------------------+     +-------------------+     +-------------------+

方案 A: Tesseract (开源本地)
  - 优点：免费、离线、隐私安全
  - 缺点：中文识别率一般、需要训练数据、部署复杂
  - 适用：内网环境、数据敏感场景

方案 B: 百度云 OCR / 阿里云 OCR (云服务)
  - 优点：识别率高、支持表格、API 简单
  - 缺点：按量收费、需要联网、数据外传
  - 适用：互联网环境、追求准确率

方案 C: 直接拒绝（本课程方案）
  - 优点：实现简单、无额外依赖
  - 缺点：用户体验稍差
  - 适用：教学场景、MVP 版本
```

**本课程的简化处理**：检测到 PDF 提取为空时，提示用户上传文本版 PDF。

```java
/**
 * 检查 PDF 是否为扫描版（图片型）
 * 原理：如果 PDFBox 提取的文本长度小于阈值，认为是扫描版
 *
 * @param extractedText PDFBox 提取出的文本
 * @return true 表示可能是扫描版，需要 OCR；false 表示是文本版 PDF
 */
public boolean isScannedPdf(String extractedText) {
    // 阈值设定：正常简历至少有 50 个字符（姓名+电话+邮箱就超过 20 个字符）
    // 如果提取结果少于 50 个字符，大概率是扫描版或空白 PDF
    return extractedText == null || extractedText.trim().length() < 50;
}
```

#### 3.3 表格布局的处理技巧

简历中常见的表格场景：
- 教育经历表格（学校 | 专业 | 时间）
- 工作经历表格（公司 | 职位 | 时间）
- 技能清单表格

PDFBox 提取表格时的问题：

```
原始表格：
+-----------+-----------+-----------+
|  学校      |   专业     |   时间    |
+-----------+-----------+-----------+
| 北京大学   |  计算机    | 2018-2022 |
+-----------+-----------+-----------+

PDFBox 提取结果（可能乱序）：
学校 专业 时间
北京大学 计算机 2018-2022

或者更糟：
学校
北京大学
专业
计算机
时间
2018-2022
```

**处理策略**：
1. 设置 `stripper.setSortByPosition(true)` 尽量保留横向顺序
2. 在 Prompt 中告诉 LLM "文本可能来自表格，请按语义理解"
3. 对于复杂表格，考虑使用专门的表格提取库（如 Tabula）

#### 3.4 中英文混排处理

简历中常见的中英文混排场景：
- 公司名："字节跳动 ByteDance"
- 技能："精通 Java、SpringBoot、Vue.js"
- 学校："清华大学 Tsinghua University"

PDFBox 对中英文混排的支持较好，但需要注意：
- 某些 PDF 生成工具会导致中英文之间插入多余空格
- 特殊符号（如 bullet point）可能变成乱码

**预处理代码**：

```java
/**
 * 清洗 PDF 提取出的原始文本
 * 处理常见问题：多余空格、特殊符号、乱码字符
 *
 * @param rawText PDFBox 提取的原始文本
 * @return 清洗后的文本，更适合 LLM 解析
 */
public String cleanExtractedText(String rawText) {
    if (rawText == null || rawText.isEmpty()) {
        return "";
    }

    String cleaned = rawText
        // 将多个连续换行符替换为两个换行符，保留段落结构但避免过多空行
        .replaceAll("\\n{3,}", "\n\n")
        // 将多个连续空格替换为单个空格
        .replaceAll(" +", " ")
        // 移除常见的乱码字符（PDF 编码问题导致的）
        .replaceAll("[\\x00-\\x08\\x0B\\x0C\\x0E-\\x1F]", "")
        // 统一中英文标点（可选，根据需求决定）
        .replace("，", ", ")
        .replace("。", ". ");

    return cleaned.trim();
}
```

### 4. Spring AI 结构化输出详解

#### 4.1 BeanOutputConverter 工作原理

`BeanOutputConverter` 是 Spring AI 提供的结构化输出工具，它的核心作用是让 LLM 按照指定的 Java 类型返回 JSON，然后自动转换为 Java 对象。

**工作原理图解**：

```
+----------------+     +-------------------+     +-------------------+
|  Java Class    | --> |   JSON Schema     | --> |    LLM Prompt     |
|  (Record/Class)|     |  (字段类型约束)    |     |  (Schema + 文本)   |
+----------------+     +-------------------+     +-------------------+
        |                                               |
        |                                               v
        |                                        +-------------------+
        |                                        |   LLM 推理输出     |
        |                                        |   (JSON 字符串)    |
        |                                        +-------------------+
        |                                               |
        v                                               v
+----------------+     +-------------------+     +-------------------+
|  Java Object   | <-- |   JSON 解析        | <-- |  BeanOutputConverter|
|  (已填充数据)   |     |  (Jackson/Gson)    |     |  .convert()       |
+----------------+     +-------------------+     +-------------------+
```

**流程说明**：
1. 你定义一个 Java Record（如 `ResumeParseResult`），包含期望的字段
2. `BeanOutputConverter` 自动生成 JSON Schema（描述每个字段的名称和类型）
3. Schema 被注入到 Prompt 中，告诉 LLM "请按这个格式返回 JSON"
4. LLM 返回 JSON 字符串
5. `BeanOutputConverter` 使用 Jackson 将 JSON 转换为 Java 对象

#### 4.2 定义解析结果 Record

```java
// ============================================
// ResumeParseResult.java - 简历解析结果 Record
// ============================================

import java.util.List; // Java 列表接口，用于存储多条记录

/**
 * 简历解析结果 Record
 * Record 是 JDK 14+ 引入的不可变数据载体，自动生成构造器、getter、equals、hashCode、toString
 * 适合作为 DTO（数据传输对象）使用
 *
 * 注意：字段名必须与 LLM 返回的 JSON 键名一致（区分大小写）
 */
public record ResumeParseResult(
    // ---------------- 基本信息 ----------------
    String name,           // 姓名，如 "张三"
    String phone,          // 手机号，如 "13800138000"
    String email,          // 邮箱，如 "zhangsan@example.com"
    Integer age,           // 年龄，Integer 表示可能为空（简历没写年龄）

    // ---------------- 学历信息 ----------------
    // List<Education> 表示多条学历记录（本科、硕士、博士等）
    List<Education> education,

    // ---------------- 工作经历 ----------------
    // List<WorkExperience> 表示多段工作经历
    List<WorkExperience> workExperience,

    // ---------------- 技能标签 ----------------
    // List<String> 表示技能列表，如 ["Java", "SpringBoot", "Vue3"]
    List<String> skills,

    // ---------------- 置信度评分 ----------------
    // 整体置信度，0.0-1.0，表示 LLM 对本次解析结果的信心程度
    Double overallConfidence
) {

    /**
     * 嵌套 Record：教育经历
     * 一条教育经历包含：学校、专业、学位、起止时间
     */
    public record Education(
        String school,      // 学校名称，如 "北京大学"
        String major,       // 专业名称，如 "计算机科学与技术"
        String degree,      // 学位，如 "本科"、"硕士"、"博士"
        String startDate,   // 开始时间，如 "2018-09"
        String endDate      // 结束时间，如 "2022-06"，可能为 "至今"
    ) {}

    /**
     * 嵌套 Record：工作经历
     * 一段工作经历包含：公司、职位、起止时间、工作描述
     */
    public record WorkExperience(
        String company,     // 公司名称，如 "阿里巴巴"
        String position,    // 职位名称，如 "高级 Java 工程师"
        String startDate,   // 开始时间，如 "2022-07"
        String endDate,     // 结束时间，如 "2024-03"，可能为 "至今"
        String description  // 工作描述，简述职责和成果
    ) {}
}
```

#### 4.3 使用 BeanOutputConverter

```java
// ============================================
// 在 Service 中使用 BeanOutputConverter
// ============================================

import org.springframework.ai.chat.client.ChatClient;           // Spring AI 聊天客户端
import org.springframework.ai.converter.BeanOutputConverter;    // 结构化输出转换器
import org.springframework.stereotype.Service;                   // Spring 服务层注解

@Service // 标记为 Spring 服务层组件，处理业务逻辑
public class ResumeParseService {

    // ChatClient 是 Spring AI 封装的大模型调用客户端
    // 通过构造器注入，已在配置类中配置好 Kimi API 参数
    private final ChatClient chatClient;

    // PDF 文本提取器，上一节定义的组件
    private final ResumePdfExtractor pdfExtractor;

    // 构造器注入依赖（Spring 推荐的方式，便于测试）
    public ResumeParseService(ChatClient chatClient, ResumePdfExtractor pdfExtractor) {
        this.chatClient = chatClient;
        this.pdfExtractor = pdfExtractor;
    }

    /**
     * 解析简历 PDF，返回结构化结果
     *
     * @param file 前端上传的 PDF 文件
     * @return ResumeParseResult 包含解析出的所有字段
     */
    public ResumeParseResult parseResume(MultipartFile file) {
        // -------- 步骤 1：提取 PDF 文本 --------
        String rawText = pdfExtractor.extractText(file);

        // 检查是否为扫描版 PDF（文本提取为空或过少）
        if (pdfExtractor.isScannedPdf(rawText)) {
            throw new IllegalArgumentException(
                "无法提取文本，该 PDF 可能是扫描版（图片）。请上传文本版 PDF。"
            );
        }

        // 清洗文本，移除多余空格和乱码
        String cleanedText = pdfExtractor.cleanExtractedText(rawText);

        // -------- 步骤 2：创建结构化输出转换器 --------
        // BeanOutputConverter 会自动生成 ResumeParseResult 的 JSON Schema
        // 并将 Schema 附加到 Prompt 中，指导 LLM 按格式输出
        BeanOutputConverter<ResumeParseResult> converter =
            new BeanOutputConverter<>(ResumeParseResult.class);

        // -------- 步骤 3：构建 Prompt 并调用 LLM --------
        // 获取自动生成的 JSON Schema（用于注入 Prompt）
        String jsonSchema = converter.getFormat();

        // 使用 ChatClient 构建对话请求
        String response = chatClient.prompt()
            // System Prompt：定义 AI 的角色和任务约束
            .system(buildSystemPrompt(jsonSchema))
            // User Prompt：传入清洗后的简历文本
            .user(cleanedText)
            // call() 同步调用 LLM，等待返回结果
            .call()
            // content() 获取 LLM 返回的文本内容（应为 JSON 字符串）
            .content();

        // -------- 步骤 4：将 JSON 转换为 Java 对象 --------
        // convert() 方法内部使用 Jackson 进行 JSON 反序列化
        // 如果 LLM 返回的 JSON 格式不正确，会抛出 ConversionException
        ResumeParseResult result = converter.convert(response);

        return result;
    }

    /**
     * 构建 System Prompt，指导 LLM 按 Schema 输出
     *
     * @param jsonSchema BeanOutputConverter 生成的 JSON Schema
     * @return 完整的 System Prompt 字符串
     */
    private String buildSystemPrompt(String jsonSchema) {
        return """
            你是一位专业的简历解析助手。你的任务是从简历文本中提取结构化信息。

            ## 输出格式要求
            请严格按照以下 JSON Schema 格式输出，不要添加任何额外说明：

            %s

            ## 字段说明
            - name: 候选人姓名
            - phone: 手机号码（11位数字）
            - email: 电子邮箱地址
            - age: 年龄（数字）
            - education: 教育经历列表，按时间倒序排列
            - workExperience: 工作经历列表，按时间倒序排列
            - skills: 技能标签列表，如 ["Java", "SpringBoot"]
            - overallConfidence: 整体置信度，0.0-1.0 之间

            ## 注意事项
            1. 如果某个字段在简历中找不到，使用 null 而不是空字符串
            2. 日期格式统一为 "YYYY-MM"
            3. 技能标签请尽量标准化（如 "Spring Boot" 统一为 "SpringBoot"）
            """.formatted(jsonSchema);
    }
}
```

#### 4.4 字段类型映射对照表

| Java 类型 | JSON 类型 | 说明 | 示例 |
|-----------|----------|------|------|
| `String` | string | 文本字段 | `"张三"` |
| `Integer` | number / null | 整数，可为空 | `25` 或 `null` |
| `Double` | number | 浮点数 | `0.95` |
| `Boolean` | boolean | 布尔值 | `true` |
| `List<String>` | array of string | 字符串列表 | `["Java", "Vue"]` |
| `List<Education>` | array of object | 对象列表 | `[{"school":"北大"}]` |
| `LocalDate` | string | 日期（需配置 Jackson） | `"2024-01-15"` |

#### 4.5 失败回退策略

LLM 有时不会严格按照 JSON Schema 输出，可能出现：
- 在 JSON 外面加了 ```json 代码块标记
- 字段名拼写错误
- 类型不匹配（如字符串写了 `"25岁"` 而不是 `25`）

**回退处理代码**：

```java
/**
 * 安全的简历解析，带有多层回退策略
 *
 * @param file 上传的 PDF 文件
     * @return 解析结果，如果所有策略都失败则返回带有错误信息的默认对象
     */
    public ResumeParseResult parseResumeSafe(MultipartFile file) {
        try {
            // 第一层：正常解析
            return parseResume(file);
        } catch (Exception e) {
            System.err.println("正常解析失败，尝试回退策略: " + e.getMessage());
        }

        try {
            // 第二层：手动清洗 LLM 输出后再解析
            String rawText = pdfExtractor.extractText(file);
            String cleanedText = pdfExtractor.cleanExtractedText(rawText);

            // 获取原始 LLM 响应
            BeanOutputConverter<ResumeParseResult> converter =
                new BeanOutputConverter<>(ResumeParseResult.class);
            String jsonSchema = converter.getFormat();

            String response = chatClient.prompt()
                .system(buildSystemPrompt(jsonSchema))
                .user(cleanedText)
                .call()
                .content();

            // 手动清洗：移除 markdown 代码块标记
            String cleanedJson = response
                .replaceAll("```json\\s*", "")  // 移除开头的 ```json
                .replaceAll("```\\s*", "")      // 移除结尾的 ```
                .trim();

            return converter.convert(cleanedJson);
        } catch (Exception e) {
            System.err.println("回退解析也失败了: " + e.getMessage());
        }

        // 第三层：返回带有错误信息的默认对象
        return new ResumeParseResult(
            "解析失败", null, null, null,
            List.of(), List.of(), List.of(), 0.0
        );
    }
```

#### 5.2 Few-shot 示例的作用

Few-shot（少样本示例）是给 LLM 看一个"输入-输出"样例，让它模仿这个模式处理新数据。

**为什么需要 Few-shot？**

| 场景 | 无 Few-shot | 有 Few-shot |
|------|------------|------------|
| 日期格式 | 可能输出 "2022年7月" | 统一输出 "2022-07" |
| 技能拆分 | 可能输出 "SpringBoot+MySQL" | 正确输出 ["SpringBoot", "MySQL"] |
| 空值处理 | 可能输出 "" 或 "未知" | 正确输出 null |
| 学历排序 | 可能正序排列 | 按示例倒序排列 |

### 6. 完整代码实现

#### 6.1 ResumeParseService（核心服务）

```java
// ============================================
// ResumeParseService.java - 简历解析服务
// ============================================

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.converter.BeanOutputConverter;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.web.multipart.MultipartFile;

import java.util.List;

@Service
public class ResumeParseService {

    private final ChatClient chatClient;
    private final ResumePdfExtractor pdfExtractor;
    private final EmpMapper empMapper;          // 员工表 Mapper
    private final EmpExprMapper empExprMapper;  // 工作经历表 Mapper

    public ResumeParseService(ChatClient chatClient,
                              ResumePdfExtractor pdfExtractor,
                              EmpMapper empMapper,
                              EmpExprMapper empExprMapper) {
        this.chatClient = chatClient;
        this.pdfExtractor = pdfExtractor;
        this.empMapper = empMapper;
        this.empExprMapper = empExprMapper;
    }

    /**
     * 解析简历（不带入库，仅返回结构化结果）
     */
    public ResumeParseResult parseResume(MultipartFile file) {
        // 1. 提取并清洗文本
        String rawText = pdfExtractor.extractText(file);
        if (pdfExtractor.isScannedPdf(rawText)) {
            throw new IllegalArgumentException("请上传文本版 PDF，扫描版暂不支持");
        }
        String cleanedText = pdfExtractor.cleanExtractedText(rawText);

        // 2. 构建转换器和 Prompt
        BeanOutputConverter<ResumeParseResult> converter =
            new BeanOutputConverter<>(ResumeParseResult.class);
        String jsonSchema = converter.getFormat();

        // 3. 调用 LLM
        String response = chatClient.prompt()
            .system(buildSystemPrompt(jsonSchema))
            .user(cleanedText)
            .call()
            .content();

        // 4. 转换结果
        return converter.convert(response);
    }

    /**
     * 确认并入库（带事务控制）
     * @Transactional 确保员工信息和工作经历要么同时入库，要么同时回滚
     */
    @Transactional(rollbackFor = Exception.class)
    public void confirmAndSave(ResumeParseResult result) {
        // 1. 创建员工实体
        Emp emp = new Emp();
        emp.setName(result.name());
        emp.setPhone(result.phone());
        emp.setEmail(result.email());
        // 其他字段映射...

        // 2. 插入员工表，获取生成的 ID
        empMapper.insert(emp);
        Long empId = emp.getId(); // MyBatis-Plus 自动回填主键

        // 3. 插入工作经历（如果有）
        if (result.workExperience() != null) {
            for (ResumeParseResult.WorkExperience we : result.workExperience()) {
                EmpExpr expr = new EmpExpr();
                expr.setEmpId(empId);           // 关联员工 ID
                expr.setCompany(we.company());   // 公司名称
                expr.setPosition(we.position()); // 职位
                expr.setStartDate(we.startDate()); // 开始时间
                expr.setEndDate(we.endDate());     // 结束时间
                expr.setDescription(we.description()); // 工作描述
                empExprMapper.insert(expr);
            }
        }

        // 4. 技能标签处理（根据你的表结构决定）
        // 可以存入扩展字段，或插入关联表
    }

    private String buildSystemPrompt(String jsonSchema) {
        return """
            你是一位专业的简历解析助手。请从简历文本中提取结构化信息。
            严格按照以下 JSON Schema 输出，不要添加任何额外说明：
            %s
            """.formatted(jsonSchema);
    }
}
```

#### 6.2 ResumeController（接口层）

```java
// ============================================
// ResumeController.java - 简历解析接口
// ============================================

import org.springframework.web.bind.annotation.*;    // REST 注解
import org.springframework.web.multipart.MultipartFile; // 文件上传
import com.tlias.common.Result;                         // 统一响应封装

@RestController // 标记为 REST 控制器，返回 JSON 而非页面
@RequestMapping("/ai/resume") // 基础路径：所有接口以 /ai/resume 开头
public class ResumeController {

    private final ResumeParseService resumeParseService;

    public ResumeController(ResumeParseService resumeParseService) {
        this.resumeParseService = resumeParseService;
    }

    /**
     * 解析简历接口
     * POST /ai/resume/parse
     *
     * @param file 前端上传的 PDF 文件（multipart/form-data 格式）
     * @return Result<ResumeParseResult> 统一响应，包含解析结果
     */
    @PostMapping("/parse")
    public Result<ResumeParseResult> parseResume(
            @RequestParam("file") MultipartFile file // @RequestParam 绑定上传的文件参数
    ) {
        // 参数校验：检查文件是否为空
        if (file.isEmpty()) {
            return Result.error("请上传 PDF 文件");
        }

        // 参数校验：检查文件类型
        String filename = file.getOriginalFilename();
        if (filename == null || !filename.toLowerCase().endsWith(".pdf")) {
            return Result.error("仅支持 PDF 格式");
        }

        // 参数校验：检查文件大小（限制 10MB）
        if (file.getSize() > 10 * 1024 * 1024) {
            return Result.error("文件大小不能超过 10MB");
        }

        try {
            // 调用服务层解析简历
            ResumeParseResult result = resumeParseService.parseResume(file);
            // 返回成功响应，data 为解析结果
            return Result.success(result);
        } catch (IllegalArgumentException e) {
            // 业务异常（如扫描版 PDF）
            return Result.error(e.getMessage());
        } catch (Exception e) {
            // 系统异常
            return Result.error("解析失败：" + e.getMessage());
        }
    }

    /**
     * 确认入库接口
     * POST /ai/resume/confirm
     *
     * @param result 前端确认后的解析结果（用户可能做了修改）
     * @return Result<String> 统一响应
     */
    @PostMapping("/confirm")
    public Result<String> confirmAndSave(
            @RequestBody ResumeParseResult result // @RequestBody 将 JSON 请求体转为 Java 对象
    ) {
        try {
            resumeParseService.confirmAndSave(result);
            return Result.success("入库成功");
        } catch (Exception e) {
            return Result.error("入库失败：" + e.getMessage());
        }
    }
}
```

#### 7.1 映射关系图解

```
ResumeParseResult (AI 解析结果)
    |
    |-- name  ------------------>  Emp.name
    |-- phone  ----------------->  Emp.phone
    |-- email  ----------------->  Emp.email
    |-- age  ------------------->  Emp.age
    |
    |-- education[]  ----------->  （根据业务需求，可扩展学历表）
    |     |-- school
    |     |-- major
    |     |-- degree
    |     |-- startDate
    |     |-- endDate
    |
    |-- workExperience[]  ------>  EmpExpr 表（多条记录）
    |     |-- company    ------->  EmpExpr.company
    |     |-- position   ------->  EmpExpr.position
    |     |-- startDate  ------->  EmpExpr.start_date
    |     |-- endDate    ------->  EmpExpr.end_date
    |     |-- description ------>  EmpExpr.description
    |
    |-- skills[]  -------------->  Emp.skills（JSON 字符串存储）
    |                             或 skill_tag 关联表
    |
    +-- overallConfidence  ----->  不入库，仅用于前端展示
```

#### 7.2 事务控制说明

`@Transactional` 注解的作用：

```
无事务的情况（危险）：
  插入 Emp 成功 --> 插入 EmpExpr1 成功 --> 插入 EmpExpr2 失败
  结果：员工存在，但工作经历不完整（数据不一致）

有事务的情况（安全）：
  插入 Emp 成功 --> 插入 EmpExpr1 成功 --> 插入 EmpExpr2 失败
  结果：所有操作回滚，员工和工作经历都不存在（数据一致）
```

```vue
// 响应式状态定义

const progress = ref(0)         // 进度条百分比 0-100
const progressStatus = ref('')  // 进度条状态：'' | 'success' | 'exception'

 * 上传前校验
 * @param file 待上传的文件对象
 * @returns boolean true 允许上传，false 阻止上传

  // 校验文件类型
  const isPdf = file.type === 'application/pdf'
  if (!isPdf) {
    ElMessage.error('仅支持 PDF 文件')
    return false
  }
  // 校验文件大小（10MB）
  const isLt10M = file.size / 1024 / 1024 < 10
  if (!isLt10M) {
    ElMessage.error('文件大小不能超过 10MB')
    return false
  }
  // 开始解析，显示进度条
  parsing.value = true
  progress.value = 30 // 上传完成，开始解析
  return true
}

/**
 * 上传成功回调
 * @param response 后端返回的响应数据
 */
const handleSuccess = (response) => {

  progressStatus.value = 'success'
  parsing.value = false

  if (response.code === 1) { // 假设 code=1 表示成功
    parseResult.value = response.data // 保存解析结果

    ElMessage.error(response.msg || '解析失败')

 * 上传失败回调

    if (res.code === 1) {
      ElMessage.success('入库成功')
      parseResult.value = null // 清空预览
    } else {
      ElMessage.error(res.msg || '入库失败')
    }
  } catch (e) {
    ElMessage.error('请求失败：' + e.message)

}
```

```vue
      </el-tag>
      <el-input
        v-model="newSkill"
        placeholder="添加技能"
        @keyup.enter="addSkill"
      />
    </div>

    <!-- 操作按钮 -->

      <el-button type="primary" @click="onConfirm">确认入库</el-button>
      <el-button @click="onCancel">取消</el-button>
    </div>
  </el-card>

<script setup>
import { ref, computed } from 'vue'

// 定义 props：接收父组件传入的解析结果

  result: { type: Object, required: true }
})

// 定义 emits：向父组件发送事件

// 创建可编辑的副本（避免直接修改 props）
const editableResult = ref(JSON.parse(JSON.stringify(props.result)))

// 置信度阈值：低于 0.7 认为是低置信度
const LOW_CONFIDENCE_THRESHOLD = 0.7

/**
 * 判断置信度是否偏低
 * @param confidence 置信度分数（0.0-1.0）
 * @returns boolean true 表示低置信度
 */
const isLowConfidence = (confidence) => {
  return confidence !== undefined && confidence < LOW_CONFIDENCE_THRESHOLD
}

/**
 * 置信度标签类型（用于 ElTag 的 type 属性）
 */
const confidenceTagType = computed(() => {
  const c = props.result.overallConfidence
  if (c >= 0.9) return 'success'
  if (c >= 0.7) return 'warning'
  return 'danger'
})

/**
 * 添加技能标签
 */
const addSkill = () => {
  if (newSkill.value.trim()) {
    editableResult.value.skills.push(newSkill.value.trim())
    newSkill.value = ''
  }
}

/**
 * 移除技能标签
 * @param index 标签索引
 */
const removeSkill = (index) => {
  editableResult.value.skills.splice(index, 1)
}

/**
 * 确认入库：发送编辑后的结果给父组件
 */
const onConfirm = () => {
  emit('confirm', editableResult.value)
}

/**
 * 取消操作
 */
const onCancel = () => {
  emit('cancel')
}
```

### 9. 边界情况处理

#### 9.1 边界情况对照表

| 边界情况 | 现象 | 检测方法 | 处理方案 |
|---------|------|---------|---------|
| 扫描版 PDF | PDFBox 提取文本为空或极少 | 文本长度 < 50 字符 | 提示用户上传文本版 PDF |
| 加密 PDF | PDFBox 抛出加密异常 | 捕获 `InvalidPasswordException` | 提示用户上传未加密 PDF |
| 字段提取不全 | 某些字段为 null | 检查返回对象字段 | 前端标红，引导用户补充 |
| 多页简历 | 文本提取正常，但内容过长 | 文本长度 > 5000 字符 | 截断或分段发送给 LLM |
| 图片简历 | 同扫描版 PDF | 文本提取为空 | 提示不支持图片简历 |
| 特殊编码 PDF | 中文乱码 | 检查是否包含乱码字符 | 尝试其他编码读取 |
| LLM 返回非 JSON | 解析异常 | 捕获 ConversionException | 回退策略：手动清洗再解析 |
| 网络超时 | LLM 调用超时 | 捕获超时异常 | 提示用户重试 |

#### 9.2 多页简历处理

```java
/**
 * 处理超长简历（多页）
 * 如果简历文本超过模型上下文长度，进行智能截断
 *
 * @param text 提取的完整文本
 * @param maxLength 最大允许长度（字符数）
 * @return 截断后的文本（保留开头和结尾，中间用省略号）
 */
public String truncateIfNeeded(String text, int maxLength) {
    if (text.length() <= maxLength) {
        return text; // 未超过限制，直接返回
    }
    // 保留前 60% 和后 40%，中间用标记分隔
    // 原理：简历的关键信息（姓名、联系方式）通常在开头，最近工作经历在结尾
    int headLength = (int) (maxLength * 0.6);
    int tailLength = (int) (maxLength * 0.4);
    return text.substring(0, headLength)
        + "\n\n[... 中间内容省略 ...]\n\n"
        + text.substring(text.length() - tailLength);
}
```

## 常见错误排查

| 阶段 | 错误现象 | 可能原因 | 排查步骤 | 解决方案 |
|------|---------|---------|---------|---------|
| **上传阶段** | 文件上传失败，返回 413 | 文件超过大小限制 | 检查 Nginx/Spring 文件大小配置 | 调整 `spring.servlet.multipart.max-file-size=10MB` |
| **上传阶段** | 上传成功但后端报 "文件为空" | 前端参数名与后端不匹配 | 检查 `@RequestParam("file")` 与前端 name 属性 | 统一参数名为 "file" |
| **提取阶段** | PDF 解析为空字符串 | 扫描版 PDF 或加密 PDF | 用 PDF 阅读器检查是否能复制文字 | 提示用户上传文本版 PDF |
| **提取阶段** | 中文乱码 | PDF 编码非 UTF-8 | 检查 PDF 生成工具 | 尝试指定编码读取 |
| **AI 阶段** | LLM 调用超时 | 网络问题或模型繁忙 | 检查 API 密钥和 base-url | 增加超时时间或重试机制 |
| **AI 阶段** | 返回内容非 JSON | Prompt 约束不够强 | 检查 LLM 原始输出 | 加强 System Prompt 的格式约束 |
| **解析阶段** | JSON 解析失败 | 字段类型不匹配 | 检查 LLM 返回的 JSON | 使用 `parseResumeSafe` 回退策略 |
| **入库阶段** | 员工插入成功但工作经历缺失 | 事务未生效或异常被吞掉 | 检查 `@Transactional` 配置 | 确保异常能正确传播触发回滚 |
| **入库阶段** | 外键约束错误 | empId 未正确回填 | 检查 MyBatis-Plus 主键回填配置 | 确认 `useGeneratedKeys="true"` |

## 本节小结

```
+----------------------------------------------------------+
|                    AI 简历解析与自动入库                    |
+----------------------------------------------------------+
|                                                          |
|  +----------------+      +----------------+              |
|  |   业务价值     | ---> |  非结构化文本   |              |
|  |  HR 效率提升   |      |  -> 结构化数据  |              |
|  +----------------+      +----------------+              |
|                                                          |
|  +----------------+      +----------------+              |
|  |   核心技术     | ---> |  PDFBox 提取   |              |
|  |                |      |  Spring AI     |              |
|  |                |      |  BeanOutputConverter          |
|  |                |      |  Prompt 工程   |              |
|  +----------------+      +----------------+              |
|                                                          |
|  +----------------+      +----------------+              |
|  |   关键设计     | ---> |  AI 解析 + 人工确认            |
|  |                |      |  置信度标红    |              |
|  |                |      |  事务入库      |              |
|  +----------------+      +----------------+              |
|                                                          |
|  +----------------+      +----------------+              |
|  |   边界处理     | ---> |  扫描版 PDF    |              |
|  |                |      |  JSON 解析失败  |              |
|  |                |      |  字段提取不全   |              |
|  +----------------+      +----------------+              |
|                                                          |
+----------------------------------------------------------+
```

**核心要点回顾**：

1. **PDF 提取**：使用 Apache PDFBox 提取文本版 PDF，扫描版需要 OCR 或提示用户重新上传
2. **结构化输出**：`BeanOutputConverter` 自动将 LLM 的 JSON 输出映射为 Java Record，减少手动解析工作
3. **Prompt 工程**：通过 System Prompt 角色定义 + Few-shot 示例 + 格式约束，显著提升解析准确率
4. **人工确认**：AI 解析不是 100%% 准确，必须设计"解析 -> 预览 -> 确认 -> 入库"的交互流程
5. **事务控制**：使用 `@Transactional` 保证员工信息和工作经历的一致性入库
6. **边界处理**：扫描版 PDF、加密 PDF、超长简历、LLM 返回异常格式都需要专门的回退策略
