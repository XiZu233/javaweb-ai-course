# 文件上传与云存储

## 学习目标

- 理解 HTTP 文件上传的原理（multipart/form-data）
- 掌握 SpringBoot 接收上传文件的方法
- 了解本地存储和阿里云 OSS 两种方案

## 核心知识点

### 1. HTTP 文件上传

文件上传使用 `multipart/form-data` 编码格式，与普通的 `application/json` 不同：

```
POST /upload HTTP/1.1
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary

------WebKitFormBoundary
Content-Disposition: form-data; name="file"; filename="avatar.jpg"
Content-Type: image/jpeg

[二进制文件内容]
------WebKitFormBoundary--
```

### 2. SpringBoot 接收文件

SpringBoot 通过 `MultipartFile` 对象接收上传文件：

```java
@Slf4j
@RestController
public class UploadController {

    @Value("${file.upload-path:./uploads/}")
    private String uploadPath;

    @PostMapping("/upload")
    public Result upload(MultipartFile file) throws IOException {
        log.info("文件上传：{}，大小：{} bytes", file.getOriginalFilename(), file.getSize());

        // 获取原始文件名
        String originalFilename = file.getOriginalFilename();
        // 截取后缀
        String extName = originalFilename.substring(originalFilename.lastIndexOf("."));
        // 生成唯一文件名（防止覆盖）
        String newFileName = UUID.randomUUID().toString() + extName;

        // 确保目录存在
        File dir = new File(uploadPath);
        if (!dir.exists()) {
            dir.mkdirs();
        }
        // 保存文件
        file.transferTo(new File(dir, newFileName));

        // 返回可访问的 URL
        return Result.success("/uploads/" + newFileName);
    }
}
```

### 3. MultipartFile 常用方法

| 方法 | 说明 |
|------|------|
| getOriginalFilename() | 获取原始文件名 |
| getSize() | 获取文件大小（字节） |
| getContentType() | 获取文件类型 |
| transferTo(File dest) | 将文件保存到目标位置 |
| getBytes() | 获取文件字节数组 |

### 4. 文件大小限制

在 `application.yml` 中配置上传限制：

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB      # 单个文件最大大小
      max-request-size: 100MB  # 整个请求最大大小
```

### 5. 阿里云 OSS 上传（生产环境）

本地存储的局限性：
- 应用重启或迁移后文件丢失
- 无法分布式共享
- 不支持 CDN 加速

生产环境通常使用对象存储服务（OSS）：

```java
@Service
public class AliyunOSSService {

    @Value("${aliyun.oss.endpoint}")
    private String endpoint;
    @Value("${aliyun.oss.accessKeyId}")
    private String accessKeyId;
    @Value("${aliyun.oss.accessKeySecret}")
    private String accessKeySecret;
    @Value("${aliyun.oss.bucketName}")
    private String bucketName;

    public String upload(MultipartFile file) throws IOException {
        OSS ossClient = new OSSClientBuilder().build(endpoint, accessKeyId, accessKeySecret);
        String fileName = UUID.randomUUID() + 
            file.getOriginalFilename().substring(file.getOriginalFilename().lastIndexOf("."));
        ossClient.putObject(bucketName, fileName, file.getInputStream());
        ossClient.shutdown();
        return "https://" + bucketName + "." + endpoint + "/" + fileName;
    }
}
```

### 6. 静态资源映射

如果使用本地存储，需要配置 SpringBoot 暴露上传目录为静态资源：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/uploads/**")
                .addResourceLocations("file:" + System.getProperty("user.dir") + "/uploads/");
    }
}
```

String contentType = file.getContentType();

## 常见错误排查

| 错误现象 | 原因 | 解决方案 |
|---------|------|---------|
| 文件上传大小超过限制 | 超出 spring.servlet.multipart.max-file-size | 调大配置或前端限制 |
| 文件保存后无法访问 | 未配置静态资源映射 | 添加 ResourceHandler |
| 文件名重复被覆盖 | 使用原始文件名保存 | 使用 UUID 重命名 |
| 中文文件名乱码 | 编码问题 | 统一使用 UTF-8 |

## 本节小结

文件上传是常见的业务需求。开发环境使用本地存储简单快捷，生产环境推荐使用阿里云 OSS 等云存储服务，保证文件持久化和高可用。
