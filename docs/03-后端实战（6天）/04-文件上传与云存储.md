# 文件上传与云存储

## 学习目标

学完本节后，你将能够：

- 理解 HTTP 文件上传的底层原理（multipart/form-data）
- 用 SpringBoot 实现文件上传接口
- 知道为什么不能用原始文件名（覆盖、安全风险）
- 区分本地存储和云存储的优缺点
- 掌握阿里云 OSS 完整接入流程
- 实现前后端联调的图片上传组件

## 核心知识点

### 1. 为什么需要文件上传——业务无处不在

#### 1.1 真实场景

```
┌──────────────────────────────────────────────────────────┐
│  日常 App 中的"文件上传"无处不在：                         │
├──────────────────────────────────────────────────────────┤
│  • 微信发图片         → 上传一张 jpg                       │
│  • 朋友圈发视频       → 上传一个 mp4                       │
│  • 钉钉发文件         → 上传一份 Excel                     │
│  • 个人头像           → 上传一张头像图                     │
│  • 简历投递           → 上传一份 PDF                       │
│  • 电商上架商品       → 上传商品图册                       │
└──────────────────────────────────────────────────────────┘
```

**Tlias 系统中的文件上传：**

- 员工头像上传
- 部门 logo 上传
- 课件资料上传
- 学员合同上传

#### 1.2 问题：JSON 装不下二进制文件

我们之前学的 `application/json` 只能传文本：

```json
{ "name": "张三", "age": 20 }
```

但**图片、视频是二进制数据**，里面有大量不可见字符（如 `\0`、`\xff` 等），不能直接放在 JSON 里。

### 2. HTTP 文件上传原理：multipart/form-data

#### 2.1 类比：邮政信封

```
普通 JSON 请求       =  写一封信
multipart 请求       =  寄一个包裹（信 + 物品）

包裹的特点：
  ┌─────────────────────────────────┐
  │ 包装盒                          │
  │  ┌───────────────────────────┐  │
  │  │ ① 收件人信息（form 字段）  │  │
  │  │ ② 纸条留言（普通参数）    │  │
  │  │ ③ 物品本身（文件）        │  │
  │  │ ④ 用胶带分隔每件物品      │  │
  │  └───────────────────────────┘  │
  └─────────────────────────────────┘
```

#### 2.2 原始 HTTP 报文长什么样

```
POST /upload HTTP/1.1
Host: localhost:8080
Content-Type: multipart/form-data; boundary=----WebKitFormBoundaryABC123
Content-Length: 12345

------WebKitFormBoundaryABC123
Content-Disposition: form-data; name="username"

张三
------WebKitFormBoundaryABC123
Content-Disposition: form-data; name="file"; filename="avatar.jpg"
Content-Type: image/jpeg

[二进制文件内容...]
------WebKitFormBoundaryABC123--
```

**关键概念：**

- `boundary`：分隔符（浏览器自动生成的随机串）
- 每个 part 用 `------xxxx` 开头
- 文本字段：`Content-Disposition: form-data; name="字段名"`
- 文件字段：多了 `filename` 和 `Content-Type`
- 整个请求以 `------xxxx--`（最后多两个 `-`）结束

**好消息：这些细节浏览器自动处理，前端代码只需用 FormData，后端用 MultipartFile。**

### 3. SpringBoot 接收文件——MultipartFile

#### 3.1 最小可运行示例

```java
package com.tlias.controller;

import com.tlias.pojo.Result;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RestController;
import org.springframework.web.multipart.MultipartFile;
import java.io.File;
import java.io.IOException;
import java.util.UUID;

@Slf4j
@RestController
public class UploadController {

    @PostMapping("/upload")
    public Result upload(MultipartFile file) throws IOException {
        // ① 打印基本信息
        log.info("文件名：{}", file.getOriginalFilename());
        log.info("文件大小：{} bytes", file.getSize());
        log.info("文件类型：{}", file.getContentType());

        // ② 提取扩展名
        String original = file.getOriginalFilename();
        String ext = original.substring(original.lastIndexOf("."));

        // ③ 用 UUID 生成唯一文件名（防止覆盖）
        String newName = UUID.randomUUID() + ext;

        // ④ 保存到本地（确保 D:/uploads/ 已存在）
        file.transferTo(new File("D:/uploads/" + newName));

        // ⑤ 返回访问 URL
        return Result.success("/uploads/" + newName);
    }
}
```

#### 3.2 用 Postman 测试

```
Method: POST
URL: http://localhost:8080/upload
Body 标签 → 选 form-data
Key: file       Type: File      Value: 选择一张图片
点击 Send
```

**期望返回：**

```json
{
    "code": 1,
    "msg": "success",
    "data": "/uploads/3a8f-...-d2e1.jpg"
}
```

#### 3.3 MultipartFile API 全表

| 方法 | 返回类型 | 说明 |
|------|---------|------|
| `getOriginalFilename()` | String | 客户端传过来的原文件名（如 `avatar.jpg`） |
| `getName()` | String | 表单字段名（如 `file`） |
| `getSize()` | long | 文件大小（字节） |
| `getContentType()` | String | MIME 类型（如 `image/jpeg`） |
| `getBytes()` | byte[] | 整个文件读为字节数组（小文件） |
| `getInputStream()` | InputStream | 文件输入流（大文件推荐） |
| `transferTo(File)` | void | 一键保存到目标文件 |
| `isEmpty()` | boolean | 是否未选择文件 |

### 4. 必须做的 5 件事

#### 4.1 文件大小限制

```yaml
# application.yml
spring:
  servlet:
    multipart:
      max-file-size: 10MB        # 单文件最大 10MB
      max-request-size: 100MB    # 整个请求最大 100MB（多文件总和）
```

**默认是 1MB，超过会报：**

```
MaxUploadSizeExceededException: Maximum upload size exceeded
```

#### 4.2 唯一文件名（UUID）

```java
// ❌ 错误：用原始文件名
file.transferTo(new File("D:/uploads/" + file.getOriginalFilename()));
// 风险：
//   ① 两个用户都上传 avatar.jpg → 后者覆盖前者
//   ② 黑客上传 ../etc/passwd → 路径穿越攻击
//   ③ 中文/空格/特殊字符 → 兼容性问题

// ✅ 正确：UUID 重命名
String ext = file.getOriginalFilename()
                  .substring(file.getOriginalFilename().lastIndexOf("."));
String newName = UUID.randomUUID() + ext;
// 例：3a8f9c12-...-d2e1.jpg
```

#### 4.3 文件类型校验（双重保险）

**第一层：MIME 类型校验**

```java
String contentType = file.getContentType();
if (contentType == null || !contentType.startsWith("image/")) {
    return Result.error("只能上传图片");
}
```

**第二层：扩展名白名单**

```java
private static final List<String> ALLOWED_EXT =
        Arrays.asList(".jpg", ".jpeg", ".png", ".gif", ".webp");

String ext = original.substring(original.lastIndexOf(".")).toLowerCase();
if (!ALLOWED_EXT.contains(ext)) {
    return Result.error("不支持的文件类型，仅支持 jpg/png/gif/webp");
}
```

**为什么要双重校验？**

`Content-Type` 由客户端发送，**可以伪造**！黑客把 `evil.exe` 改名为 `evil.jpg`，Content-Type 设成 `image/jpeg` 就能绕过单层校验。生产环境推荐再加第三层：**文件魔数校验**（读取文件前几个字节）。

#### 4.4 文件大小校验

```java
long maxSize = 5 * 1024 * 1024;  // 5MB
if (file.getSize() > maxSize) {
    return Result.error("文件不能超过 5MB");
}
```

#### 4.5 异常处理

```java
@PostMapping("/upload")
public Result upload(MultipartFile file) {
    if (file == null || file.isEmpty()) {
        return Result.error("请选择文件");
    }
    try {
        // ...上传逻辑
        return Result.success(url);
    } catch (IOException e) {
        log.error("文件上传失败", e);
        return Result.error("上传失败，请重试");
    }
}
```

### 5. 本地存储——开发环境最简方案

#### 5.1 完整 Controller

```java
@Slf4j
@RestController
public class UploadController {

    @Value("${file.upload-path:./uploads/}")
    private String uploadPath;  // 从配置读取上传目录

    private static final List<String> ALLOWED_EXT =
            Arrays.asList(".jpg", ".jpeg", ".png", ".gif");

    private static final long MAX_SIZE = 10 * 1024 * 1024;  // 10MB

    @PostMapping("/upload")
    public Result upload(MultipartFile file) {
        // 1. 校验
        if (file == null || file.isEmpty()) {
            return Result.error("请选择文件");
        }
        if (file.getSize() > MAX_SIZE) {
            return Result.error("文件不能超过 10MB");
        }

        String original = file.getOriginalFilename();
        if (original == null || !original.contains(".")) {
            return Result.error("文件名不合法");
        }

        String ext = original.substring(original.lastIndexOf(".")).toLowerCase();
        if (!ALLOWED_EXT.contains(ext)) {
            return Result.error("仅支持 jpg/png/gif");
        }

        try {
            // 2. 创建目录
            File dir = new File(uploadPath);
            if (!dir.exists() && !dir.mkdirs()) {
                throw new IOException("创建目录失败");
            }

            // 3. 生成新文件名
            String newName = UUID.randomUUID() + ext;

            // 4. 保存文件
            file.transferTo(new File(dir, newName));

            log.info("文件保存成功：{}", newName);
            return Result.success("/uploads/" + newName);
        } catch (IOException e) {
            log.error("文件上传失败", e);
            return Result.error("上传失败");
        }
    }
}
```

#### 5.2 静态资源映射——让浏览器能访问到

仅仅保存了文件还不够，浏览器访问 `http://localhost:8080/uploads/xxx.jpg` 会 404。**需要把文件目录暴露成静态资源：**

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Value("${file.upload-path:./uploads/}")
    private String uploadPath;

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // URL 路径 → 文件系统路径
        registry.addResourceHandler("/uploads/**")
                .addResourceLocations("file:" + new File(uploadPath).getAbsolutePath() + "/");
    }
}
```

**配置后：**

```
URL：http://localhost:8080/uploads/xxx.jpg
对应：D:/uploads/xxx.jpg
```

#### 5.3 本地存储的局限性

```
┌───────────────────────────────────────────────────────────────────┐
│              本地存储 vs 云存储 对比                                │
├───────────────────────────────────────────────────────────────────┤
│  问题点              本地存储        云存储                         │
│  ──────────────────  ────────────    ──────────────                │
│  服务重启            文件还在         文件还在                       │
│  服务器换机          文件丢失❌       文件还在                       │
│  多机部署            文件不共享❌     全机共享                       │
│  CDN 加速            不支持❌         天然支持                       │
│  容量上限            硬盘大小         几乎无限                       │
│  HTTPS               需自配证书       自带证书                       │
│  防盗链/水印         需自实现         自带功能                       │
│  成本                免费             按流量计费                     │
│                                                                    │
│  结论：开发用本地，生产用云存储                                       │
└───────────────────────────────────────────────────────────────────┘
```

### 6. 阿里云 OSS 上传——生产环境标配

#### 6.1 OSS 是什么

OSS = Object Storage Service（对象存储服务）

**类比：**

- 一个无限大的网络硬盘
- 每个文件叫一个"对象"
- 文件按"桶（bucket）"分组
- 通过 URL 直接访问

#### 6.2 准备工作

**步骤 1：开通 OSS 服务**

1. 登录阿里云控制台 → 对象存储 OSS
2. 开通服务（注意：流量和存储都按量计费）
3. 创建 Bucket（推荐"标准存储"+"公共读"）

**步骤 2：获取 AccessKey**

1. 控制台右上角 → AccessKey 管理
2. 创建 RAM 子账号（**不要用主账号 AccessKey！**）
3. 给子账号授权 `AliyunOSSFullAccess` 权限
4. 记下 `accessKeyId` 和 `accessKeySecret`

**步骤 3：找到 Endpoint**

```
华东1（杭州）：oss-cn-hangzhou.aliyuncs.com
华北2（北京）：oss-cn-beijing.aliyuncs.com
（Bucket 概览页可见）
```

#### 6.3 引入依赖

```xml
<dependency>
    <groupId>com.aliyun.oss</groupId>
    <artifactId>aliyun-sdk-oss</artifactId>
    <version>3.17.4</version>
</dependency>
```

#### 6.4 配置文件

```yaml
# application.yml
aliyun:
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    access-key-id: ${ALIYUN_KEY_ID}              # 用环境变量，不要硬编码
    access-key-secret: ${ALIYUN_KEY_SECRET}
    bucket-name: tlias-demo
```

**⚠️ 安全提示：AccessKey 千万不要提交到 Git！** 应该用环境变量或者配置中心，详见后面"安全注意事项"。

#### 6.5 配置类

```java
@Data
@Component
@ConfigurationProperties(prefix = "aliyun.oss")
public class AliyunOSSProperties {
    private String endpoint;
    private String accessKeyId;
    private String accessKeySecret;
    private String bucketName;
}
```

#### 6.6 上传 Service

```java
@Slf4j
@Service
public class AliyunOSSService {

    @Autowired
    private AliyunOSSProperties props;

    public String upload(MultipartFile file) throws IOException {
        // ① 创建 OSS 客户端
        OSS ossClient = new OSSClientBuilder()
                .build(props.getEndpoint(),
                       props.getAccessKeyId(),
                       props.getAccessKeySecret());

        try {
            // ② 生成对象名（路径 + 唯一名）
            String original = file.getOriginalFilename();
            String ext = original.substring(original.lastIndexOf("."));
            String objectName = "avatar/" + UUID.randomUUID() + ext;

            // ③ 上传（用流，避免大文件 OOM）
            ossClient.putObject(props.getBucketName(),
                                objectName,
                                file.getInputStream());

            // ④ 拼接访问 URL
            String url = "https://" + props.getBucketName() + "."
                       + props.getEndpoint() + "/" + objectName;
            log.info("上传成功：{}", url);
            return url;
        } finally {
            ossClient.shutdown();   // 关闭连接，释放资源
        }
    }
}
```

#### 6.7 Controller

```java
@RestController
public class UploadController {

    @Autowired
    private AliyunOSSService ossService;

    @PostMapping("/upload")
    public Result upload(MultipartFile file) throws IOException {
        String url = ossService.upload(file);
        return Result.success(url);
    }
}
```

#### 6.8 安全注意事项

| 问题 | 解决方案 |
|------|---------|
| AccessKey 不能硬编码 | 用环境变量、配置中心或 Nacos |
| AccessKey 不能提交到 Git | `.gitignore` 加入 `application-prod.yml`，仓库只放 `application-dev.yml` 模板 |
| 客户端直传更安全 | 后端只生成"临时签名"，文件不经过后端，参考"STS 临时凭证" |
| Bucket 不要全设为公共写 | 仅 `公共读` 即可，`公共读写` 易被攻击者上传违规内容 |

### 7. 前端集成（Vue 3 + Element Plus）

#### 7.1 简单上传

```vue
<script setup>
import { ref } from 'vue';

const imageUrl = ref('');

const onSuccess = (response) => {
    imageUrl.value = response.data;  // 后端返回的 URL
};

const beforeUpload = (file) => {
    // 前端预校验
    const isImg = file.type.startsWith('image/');
    const isLt5M = file.size / 1024 / 1024 < 5;
    if (!isImg) {
        ElMessage.error('只能上传图片');
        return false;
    }
    if (!isLt5M) {
        ElMessage.error('图片不能超过 5MB');
        return false;
    }
    return true;
};
</script>

<template>
    <el-upload
        action="http://localhost:8080/upload"
        :show-file-list="false"
        :on-success="onSuccess"
        :before-upload="beforeUpload"
    >
        <img v-if="imageUrl" :src="imageUrl" class="avatar" />
        <el-icon v-else><Plus /></el-icon>
    </el-upload>
</template>

<style scoped>
.avatar { width: 100px; height: 100px; }
</style>
```

## 常见错误排查

### 阶段 1：依赖与配置问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `MultipartException` 启动报错 | 配置错位 | 检查 `spring.servlet.multipart` 缩进 |
| 找不到 OSS 类 | 没引依赖 | pom.xml 添加 aliyun-sdk-oss |
| `IllegalState: No qualifying bean...AliyunOSSProperties` | 没加 `@Component` | 配置类加 `@Component` 和 `@ConfigurationProperties` |

### 阶段 2：上传请求问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| 400 Bad Request | Content-Type 不对 | Postman 用 `form-data`，不是 `raw json` |
| `Required request part 'file' is not present` | 字段名不匹配 | 前端 FormData key 与 `MultipartFile` 参数名一致 |
| `Maximum upload size exceeded` | 文件超出限制 | 调大 `max-file-size` |
| 中文文件名乱码 | 编码问题 | 用 UUID 重命名，避免直接用原始名 |

### 阶段 3：保存与访问问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `transferTo` 报"目录不存在" | 父目录没创建 | 先 `dir.mkdirs()` |
| 保存后浏览器 404 | 没配静态资源映射 | WebConfig 添加 ResourceHandler |
| 保存路径用 `./uploads` 但找不到 | 相对路径基于工作目录 | 改用绝对路径或 `new File(...).getAbsolutePath()` |

### 阶段 4：阿里云 OSS 问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `InvalidAccessKeyId` | AccessKey 错误或被禁用 | 检查 RAM 控制台，重新生成 |
| `AccessDenied` | 没有权限 | 给 RAM 用户授 OSS 权限 |
| `NoSuchBucket` | Bucket 名错误 | 控制台核对 Bucket 名 |
| 返回的 URL 浏览器 403 | Bucket 是私有读 | 改成"公共读"，或用签名 URL |
| `SocketTimeoutException` | endpoint 错误或网络限制 | 检查 endpoint 与 Bucket 所在地域是否一致 |

│    ③ 类型校验（MIME + 扩展名双层）                                    │
│    ④ 大小校验                                                        │
│    ⑤ 异常处理                                                        │
│                                                                     │
│  本地存储：                                                          │
│    保存文件 + 配置静态资源映射                                        │
│    优点：简单                                                        │
│    缺点：换机丢失、不能集群                                           │
│                                                                     │
│  阿里云 OSS（生产推荐）：                                            │
│    ① 创建 Bucket                                                    │
│    ② 创建 RAM 子账号 + 授权                                          │
│    ③ 引入 SDK + 配置 endpoint/key                                   │
│    ④ ossClient.putObject() 上传                                      │
│    ⑤ 拼接 URL 返回                                                   │
│                                                                     │
│  安全红线：                                                          │
│    • AccessKey 不写代码、不进 Git → 用环境变量                        │
│    • Bucket 设为"公共读"，不要"公共读写"                              │
│    • 推荐 STS 临时凭证 + 客户端直传                                   │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
