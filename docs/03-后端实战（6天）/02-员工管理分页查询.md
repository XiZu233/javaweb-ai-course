## 核心知识点

### 1. 为什么需要分页——"一次拉一吨数据"是灾难

#### 1.1 不分页的问题

想象一个公司有 10 万员工：

```
不分页查询：
┌──────────────────────────────────────────────────────────┐
│  SELECT * FROM emp;  → 一次返回 10 万条数据              │
├──────────────────────────────────────────────────────────┤
│  ① 数据库压力大：扫描整张表，CPU 飙升                    │
│  ② 网络传输慢：10 万条 JSON 数据，可能几十 MB            │
│  ③ 前端崩溃：浏览器要渲染 10 万行表格，直接卡死          │
│  ④ 用户体验差：用户根本看不完 10 万行数据                │
└──────────────────────────────────────────────────────────┘
```

#### 1.2 分页的解决方案

把数据切成"一页一页"，每次只返回当前页：

```
分页查询：
  第 1 页（10 条）→ 第 2 页（10 条）→ ... → 第 10000 页（10 条）

用户只看当前页：
  ┌─────────────────────┐
  │ 第 1 页（10 条）     │
  │ ─────────────────── │
  │ 张三  研发  2020-01 │
  │ 李四  市场  2020-02 │
  │ ...                 │
  │ ─────────────────── │
  │ < 1 2 3 ... 10000 > │  ← 分页器
  └─────────────────────┘
```

**关键收益：**
- 数据库只需查询 10 条 → 响应快
- 网络只传 10 条 → 流量小
- 前端只渲染 10 条 → 不卡顿

#### 1.3 SQL 层面的分页原理

MySQL 用 `LIMIT` 实现分页：

```sql
-- 查询第 1 页，每页 10 条
SELECT * FROM emp LIMIT 0, 10;
-- LIMIT offset, count
-- offset = (页码 - 1) × 每页数量

-- 查询第 2 页，每页 10 条
SELECT * FROM emp LIMIT 10, 10;

-- 查询第 3 页，每页 10 条
SELECT * FROM emp LIMIT 20, 10;
```

**计算公式：**
```
offset（跳过条数） = (page - 1) × pageSize
count（取多少条） = pageSize
```

### 2. 使用 PageHelper 简化分页

#### 2.1 为什么用 PageHelper

如果手动写分页，每个查询都要：
1. 算 offset
2. 写 LIMIT
3. 再写一条 COUNT 查询统计总数
4. 封装结果

**PageHelper 把这些都自动做了，只需一行代码。**

#### 2.2 引入依赖

```xml
<!-- pom.xml -->
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper-spring-boot-starter</artifactId>
    <version>1.4.7</version>
</dependency>
```

#### 2.3 配置（可选）

```yaml
# application.yml
pagehelper:
  helper-dialect: mysql       # 数据库方言
  reasonable: true            # 启用合理化（page<1 时返回第1页，page>总页数时返回最后一页）
  support-methods-arguments: true
  params: count=countSql
```

### 3. 实现分页查询

#### 3.1 业务需求

员工管理需要支持：
- 分页查询（默认每页 10 条）
- 按姓名模糊搜索
- 按性别筛选
- 按入职日期范围筛选

#### 3.2 数据库表

```sql
CREATE TABLE emp (
    id          INT          PRIMARY KEY AUTO_INCREMENT,
    username    VARCHAR(20)  NOT NULL UNIQUE  COMMENT '用户名',
    password    VARCHAR(32)  DEFAULT '123456' COMMENT '密码',
    name        VARCHAR(10)  NOT NULL         COMMENT '姓名',
    gender      TINYINT(1)                    COMMENT '性别：1男 2女',
    image       VARCHAR(300)                  COMMENT '头像 URL',
    job         TINYINT(1)                    COMMENT '职位：1班主任 2讲师 3学工主管 4教研主管',
    entrydate   DATE                          COMMENT '入职日期',
    dept_id     INT                           COMMENT '部门 ID',
    create_time DATETIME    DEFAULT NOW(),
    update_time DATETIME    DEFAULT NOW()
);
```

#### 3.3 实体类

```java
package com.tlias.pojo;

import com.fasterxml.jackson.annotation.JsonFormat;
import lombok.Data;
import java.time.LocalDate;
import java.time.LocalDateTime;

@Data
public class Emp {
    private Integer id;
    private String username;
    private String password;
    private String name;
    private Integer gender;
    private String image;
    private Integer job;

    @JsonFormat(pattern = "yyyy-MM-dd")  // 日期格式化
    private LocalDate entrydate;

    private Integer deptId;

    @JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private LocalDateTime createTime;
    private LocalDateTime updateTime;

    private String deptName;  // 关联查询，部门名（不在 emp 表中）
}
```

#### 3.4 Mapper 层（用 XML 写动态 SQL）

```java
// EmpMapper.java
@Mapper
public interface EmpMapper {
    /**
     * 分页查询员工列表
     * @param name 姓名（模糊匹配）
     * @param gender 性别
     * @param begin 入职日期开始
     * @param end 入职日期结束
     */
    List<Emp> list(@Param("name") String name,
                   @Param("gender") Integer gender,
                   @Param("begin") LocalDate begin,
                   @Param("end") LocalDate end);
}
```

```xml
<!-- src/main/resources/mapper/EmpMapper.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.tlias.mapper.EmpMapper">

    <select id="list" resultType="com.tlias.pojo.Emp">
        SELECT
            e.*, d.name AS dept_name
        FROM emp e
        LEFT JOIN dept d ON e.dept_id = d.id
        <where>
            <if test="name != null and name != ''">
                AND e.name LIKE CONCAT('%', #{name}, '%')
            </if>
            <if test="gender != null">
                AND e.gender = #{gender}
            </if>
            <if test="begin != null">
                AND e.entrydate &gt;= #{begin}
            </if>
            <if test="end != null">
                AND e.entrydate &lt;= #{end}
            </if>
        </where>
        ORDER BY e.update_time DESC
    </select>

</mapper>
```

**XML 中的特殊字符：**
- `>` 在 XML 中要写 `&gt;`
- `<` 在 XML 中要写 `&lt;`
- 或者用 `<![CDATA[ ... ]]>` 包裹

#### 3.5 Service 层

```java
package com.tlias.service.impl;

import com.github.pagehelper.PageHelper;
import com.github.pagehelper.PageInfo;
import com.tlias.mapper.EmpMapper;
import com.tlias.pojo.Emp;
import com.tlias.service.EmpService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import java.time.LocalDate;
import java.util.List;

@Service
public class EmpServiceImpl implements EmpService {

    @Autowired
    private EmpMapper empMapper;

    @Override
    public PageInfo<Emp> page(Integer page, Integer pageSize,
                              String name, Integer gender,
                              LocalDate begin, LocalDate end) {
        // ① 启动 PageHelper（紧跟下一条 SELECT）
        PageHelper.startPage(page, pageSize);

        // ② 执行查询（PageHelper 会自动加 LIMIT，并额外执行 COUNT）
        List<Emp> list = empMapper.list(name, gender, begin, end);

        // ③ 用 PageInfo 包装结果（包含 total、list、pageNum 等信息）
        return new PageInfo<>(list);
    }
}
```

#### 3.6 Controller 层

```java
package com.tlias.controller;

import com.github.pagehelper.PageInfo;
import com.tlias.pojo.Emp;
import com.tlias.pojo.Result;
import com.tlias.service.EmpService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.format.annotation.DateTimeFormat;
import org.springframework.web.bind.annotation.*;
import java.time.LocalDate;

@RestController
@RequestMapping("/emps")
public class EmpController {

    @Autowired
    private EmpService empService;

    /**
     * 分页 + 条件查询
     * GET /emps?page=1&pageSize=10&name=张&gender=1&begin=2020-01-01&end=2024-12-31
     */
    @GetMapping
    public Result page(
            @RequestParam(defaultValue = "1") Integer page,
            @RequestParam(defaultValue = "10") Integer pageSize,
            @RequestParam(required = false) String name,
            @RequestParam(required = false) Integer gender,
            @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate begin,
            @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd") LocalDate end
    ) {
        PageInfo<Emp> pageInfo = empService.page(page, pageSize, name, gender, begin, end);
        return Result.success(pageInfo);
    }
}
```

**`@RequestParam` 用法：**

| 用法 | 说明 |
|------|------|
| `@RequestParam Integer page` | 必填参数，缺失会报错 |
| `@RequestParam(required=false) String name` | 选填参数 |
| `@RequestParam(defaultValue="1") Integer page` | 选填，未传时为 1 |
| `@DateTimeFormat(pattern="yyyy-MM-dd")` | 字符串自动转 LocalDate |

### 4. 返回数据结构

调用 `GET /emps?page=1&pageSize=10` 返回：

```json
{
    "code": 1,
    "msg": "success",
    "data": {
        "total": 85,                    // 总记录数
        "list": [                       // 当前页数据
            { "id": 1, "name": "张三", ... },
            { "id": 2, "name": "李四", ... },
            ... 共 10 条
        ],
        "pageNum": 1,                   // 当前页码
        "pageSize": 10,                 // 每页条数
        "pages": 9,                     // 总页数 = ceil(total / pageSize)
        "hasNextPage": true,            // 有下一页吗
        "hasPreviousPage": false,       // 有上一页吗
        "isFirstPage": true,            // 是第一页吗
        "isLastPage": false,            // 是最后一页吗
        "navigatePages": 8,             // 导航页数
        "navigatepageNums": [1,2,3,4,5,6,7,8]
    }
}
```

#### 4.1 PageInfo 常用属性详解

| 属性 | 说明 | 用途 |
|------|------|------|
| total | 总记录数 | 显示"共 85 条" |
| list | 当前页数据 | 渲染表格 |
| pageNum | 当前页码 | 高亮当前页 |
| pageSize | 每页条数 | 显示分页条 |
| pages | 总页数 | 显示"第 1/9 页" |
| hasNextPage | 是否有下一页 | "下一页"按钮是否可点 |
| hasPreviousPage | 是否有上一页 | "上一页"按钮是否可点 |

### 5. PageHelper 使用陷阱

#### 5.1 startPage 必须紧跟查询

```java
// ✅ 正确：startPage 后紧跟需要分页的查询
PageHelper.startPage(page, pageSize);
List<Emp> emps = empMapper.list(...);  // 这条被分页

// ❌ 错误：中间穿插其他查询
PageHelper.startPage(page, pageSize);
List<Dept> depts = deptMapper.list();  // 这条被分页了！
List<Emp> emps = empMapper.list(...);  // 这条没有分页
```

#### 5.2 分页只对第一个查询生效

```java
// ❌ 错误
PageHelper.startPage(page, pageSize);
List<Emp> emps1 = empMapper.list(...);  // 这条被分页
List<Emp> emps2 = empMapper.list(...);  // 这条没有分页

// ✅ 正确：每次都重新调用 startPage
PageHelper.startPage(page, pageSize);
List<Emp> emps1 = empMapper.list(...);

PageHelper.startPage(page, pageSize);
List<Emp> emps2 = empMapper.list(...);
```

#### 5.3 ThreadLocal 内存泄漏

PageHelper 用 ThreadLocal 传递分页参数。如果调用 `startPage` 后没有执行查询（如抛异常），分页参数会残留在 ThreadLocal 中。

**最佳实践：** PageHelper 5.0+ 会自动清理，无需手动处理。

### 6. 前后端联调示例

#### 6.1 前端调用（Vue + Axios）

```javascript
// api/emp.js
import request from '@/utils/request';

export const getEmpList = (params) => request.get('/emps', { params });
```

```vue
<script setup>
import { ref, onMounted } from 'vue';
import { getEmpList } from '@/api/emp';

const tableData = ref([]);
const total = ref(0);
const queryForm = ref({
    page: 1,
    pageSize: 10,
    name: '',
    gender: null,
    begin: '',
    end: ''
});

const fetchData = async () => {
    const res = await getEmpList(queryForm.value);
    tableData.value = res.list;
    total.value = res.total;
};

const handlePageChange = (newPage) => {
    queryForm.value.page = newPage;
    fetchData();
};

onMounted(fetchData);
</script>

<template>
    <el-table :data="tableData">
        <el-table-column prop="name" label="姓名" />
        <el-table-column prop="gender" label="性别" />
        <el-table-column prop="entrydate" label="入职日期" />
    </el-table>

    <el-pagination
        :current-page="queryForm.page"
        :page-size="queryForm.pageSize"
        :total="total"
        @current-change="handlePageChange"
    />
</template>
```

### 练习 2：分页参数边界保护

要求：
- page < 1 时，按 page=1 处理
- pageSize > 100 时，按 pageSize=100 处理（防止恶意大分页拖垮数据库）

```java
@Override
public PageInfo<Emp> page(Integer page, Integer pageSize, ...) {
    // 边界保护
    if (page == null || page < 1) page = 1;
    if (pageSize == null || pageSize < 1) pageSize = 10;
    if (pageSize > 100) pageSize = 100;

    PageHelper.startPage(page, pageSize);
    ...
}
```

## 常见错误排查

### 阶段 1：依赖问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `PageHelper class not found` | 依赖没引入 | pom.xml 添加 pagehelper-spring-boot-starter |
| 分页不生效（返回所有数据） | 未启用 PageHelper | 用 starter 自动配置，或 application.yml 配置 dialect |

### 阶段 2：参数问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `Required parameter not present` | 没加 `required=false` | 选填参数加 `required=false` |
| 日期参数报错 | 字符串无法转 LocalDate | 加 `@DateTimeFormat(pattern="yyyy-MM-dd")` |
| 中文搜索没结果 | 编码问题 | 数据库用 utf8mb4，URL 编码 |

### 阶段 3：SQL 问题

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| `&gt; / &lt;` 报错 | XML 没转义 | `>` 写 `&gt;`，`<` 写 `&lt;` |
| 模糊查询 like 不生效 | 没加 % | `LIKE CONCAT('%', #{name}, '%')` |
| 时间范围查询不准 | 没考虑当天 23:59:59 | 用 `&lt;=` 或加 1 天 |

### 阶段 4：分页结果异常

| 错误现象 | 原因分析 | 解决方案 |
|---------|---------|---------|
| total 不对 | 中间有其他查询消费了分页 | startPage 紧跟目标查询 |
| 第 1 页正常，第 2 页空 | 数据不足 | 检查总数据量 |
| 分页插件抛异常 | 多次启动分页未消费 | 清理 ThreadLocal 或检查代码逻辑 |
