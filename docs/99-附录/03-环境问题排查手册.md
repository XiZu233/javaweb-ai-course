# 附录三：环境问题排查手册

> 适用版本：Version A / Version B 全栈开发环境
> 目标：独立解决开发过程中 90% 以上的环境配置问题，减少因环境问题导致的学习中断

---

## 一、概述与目标

本文档汇总了 Tlias 人事管理系统开发过程中最常见的环境问题及解决方案，涵盖以下学习目标：

1. **独立完成 Java 环境配置**：理解 JDK、JRE、JAVA_HOME 的关系，能处理多版本切换
2. **解决 Maven 依赖问题**：配置国内镜像、处理依赖冲突、清理本地仓库
3. **排查数据库连接问题**：MySQL 安装、字符集、时区、权限配置
4. **处理 IDE 常见问题**：IDEA 编译版本、Lombok 插件、缓存清理
5. **解决前端环境故障**：Node 版本管理、npm 镜像、端口占用
6. **Docker 与网络问题**：容器启动、端口映射、WSL2、跨域

## 二、Java 环境问题

### 2.1 什么是 JDK？为什么要配置环境变量？

JDK（Java Development Kit）是 Java 开发工具包，包含编译器（javac）、运行时（JRE）和各种工具。配置环境变量的目的是让操作系统在任何目录下都能找到 Java 命令。

```
JDK 组成结构：
+-----------------------------+
|           JDK               |
|  +-----------------------+  |
|  |         JRE           |  |
|  |  +-----------------+  |  |
|  |  |      JVM        |  |  |
|  |  +-----------------+  |  |
|  |  | 核心类库(rt.jar) |  |  |
|  |  +-----------------+  |  |
|  +-----------------------+  |
|  | 编译器 javac          |  |
|  | 调试器 jdb            |  |
|  | 文档工具 javadoc      |  |
|  +-----------------------+  |
+-----------------------------+
```

### 2.2 JDK 安装与验证

```bash
# 1. 验证 Java 是否已安装
java -version

# 预期输出（Version A 使用 JDK 8）：
# java version "1.8.0_xxx"
# Java(TM) SE Runtime Environment (build 1.8.0_xxx)
# Java HotSpot(TM) 64-Bit Server VM (build 25.xxx, mixed mode)

# 预期输出（Version B 使用 JDK 17）：
# openjdk version "17.0.x" 202x-xx-xx
# OpenJDK Runtime Environment (build 17.0.x+xx)
# OpenJDK 64-Bit Server VM (build 17.0.x+xx, mixed mode, sharing)

# 2. 验证 javac 编译器
javac -version

# 3. 查看 JAVA_HOME 是否配置
# Windows PowerShell:
$env:JAVA_HOME

# Windows CMD:
echo %JAVA_HOME%
```

### 2.3 JAVA_HOME 环境变量配置（Windows）

```
配置步骤：
+----------------------------------------------------------+
| 1. 右键"此电脑" -> 属性 -> 高级系统设置 -> 环境变量      |
| 2. 系统变量 -> 新建                                       |
|    变量名：JAVA_HOME                                     |
|    变量值：C:\Program Files\Java\jdk-17                  |
| 3. 编辑 Path 变量，添加：%JAVA_HOME%\bin                 |
| 4. 保存后，新开 CMD/PowerShell 验证                       |
+----------------------------------------------------------+
```

### 2.4 多版本 JDK 切换

```bash
# 查看已安装的所有 JDK 版本（Windows 手动切换）
# 修改 JAVA_HOME 指向不同目录即可

# 临时切换（当前终端有效）
# PowerShell:
$env:JAVA_HOME = "C:\Program Files\Java\jdk-17"
$env:Path = "$env:JAVA_HOME\bin;$env:Path"
java -version

# 使用 jEnv（Mac/Linux 推荐）
jenv versions          # 列出所有版本
jenv global 17         # 全局切换
jenv local 1.8         # 当前目录切换
```

### 2.5 Java 环境常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `'java' 不是内部或外部命令` | 未配置环境变量或 Path 错误 | 检查 JAVA_HOME 和 Path 配置；新开终端 |
| `UnsupportedClassVersionError` | 编译和运行 JDK 版本不一致 | 统一使用同一版本；检查 IDEA 项目 SDK 设置 |
| `Could not reserve enough space` | 32位 JDK 内存不足 | 安装 64位 JDK |
| `Error: Could not find or load main class` | 类路径错误或包声明不匹配 | 检查包名和目录结构是否一致 |

## 三、Maven 问题

### 3.1 什么是 Maven？为什么要配置镜像？

Maven 是 Java 项目的依赖管理和构建工具。它从中央仓库下载依赖（如 SpringBoot、MyBatis），但中央仓库在国外，国内访问慢且不稳定。配置阿里云镜像可以大幅提升依赖下载速度。

```
Maven 依赖下载流程：
+-----------+      +----------------+      +------------------+
| 你的项目   |  --> | settings.xml   |  --> | 阿里云镜像仓库    |
| pom.xml   |      | (镜像配置)      |      | (国内加速)       |
+-----------+      +----------------+      +------------------+
       |                                          |
       | (如果镜像没有)                              |
       v                                          v
+-----------+      +----------------+      +------------------+
| 本地仓库   | <--  | 阿里云仓库     | <--  | Maven 中央仓库   |
| .m2/repo  |      | (国内缓存)      |      | (国外源)         |
+-----------+      +----------------+      +------------------+
```

### 3.2 配置阿里云镜像（settings.xml）

```xml
<!-- 文件位置：C:\Users\你的用户名\.m2\settings.xml -->
<!-- 如果没有此文件，手动创建 -->
<settings>
    <mirrors>
        <!-- 阿里云镜像（推荐） -->
        <mirror>
            <id>aliyun</id>
            <name>Aliyun Maven</name>
            <url>https://maven.aliyun.com/repository/public</url>
            <mirrorOf>central</mirrorOf>
        </mirror>

        <!-- 华为云镜像（备选） -->
        <mirror>
            <id>huawei</id>
            <name>Huawei Maven</name>
            <url>https://repo.huaweicloud.com/repository/maven/</url>
            <mirrorOf>central</mirrorOf>
        </mirror>
    </mirrors>

    <!-- 配置本地仓库路径（可选，默认在 C:\Users\xxx\.m2\repository） -->
    <localRepository>D:/maven-repo</localRepository>
</settings>
```

### 3.3 依赖下载失败排查

```bash
# 1. 查看具体错误信息
# 在 IDEA 的 Maven 面板中查看报错详情

# 2. 删除有问题的依赖，强制重新下载
# 找到本地仓库中对应文件夹，删除后刷新 Maven
# 例如 SpringBoot 依赖有问题：
rm -rf ~/.m2/repository/org/springframework/boot

# Windows PowerShell:
Remove-Item -Recurse -Force "$env:USERPROFILE\.m2\repository\org\springframework\boot"

# 3. 在 IDEA 中重新加载 Maven 项目
# 右键 pom.xml -> Maven -> Reload Project
# 或点击 Maven 面板中的刷新按钮

# 4. 命令行强制更新
mvn clean install -U
# -U 参数：强制更新快照依赖
```

### 3.4 Maven 版本冲突解决

```
版本冲突原理：
+----------------------------------------------------------+
| 项目依赖 A-v1.0                                          |
|    └── 依赖 B-v1.0                                       |
|                                                          |
| 项目依赖 C-v2.0                                          |
|    └── 依赖 B-v2.0                                       |
|                                                          |
| 问题：B 有两个版本，Maven 默认使用"就近原则"               |
| 解决：在 pom.xml 中显式声明 B 的版本                      |
+----------------------------------------------------------+
```

```xml
<!-- 在 pom.xml 中显式声明依赖版本，覆盖传递依赖 -->
<dependency>
    <groupId>com.example</groupId>
    <artifactId>library-b</artifactId>
    <version>2.0.0</version>  <!-- 显式指定版本 -->
</dependency>
```

```bash
# 查看依赖树，分析版本冲突
mvn dependency:tree

# 过滤查看某个依赖
mvn dependency:tree -Dincludes=com.example:library-b
```

### 3.5 Maven 常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `Could not find artifact` | 依赖不存在或仓库无法访问 | 检查依赖坐标；确认镜像配置正确 |
| `Cannot access aliyun repo` | 网络问题或镜像 URL 错误 | 检查网络；确认 URL 为 https://maven.aliyun.com/repository/public |
| `Failed to read artifact descriptor` | 本地仓库文件损坏 | 删除对应文件夹重新下载 |
| `No compiler is provided` | 未配置 JDK，只有 JRE | IDEA 中设置 Project SDK 为 JDK |
| `Plugin not found` | Maven 版本过低 | 升级 Maven 到 3.6+ |
| `OutOfMemoryError` | 内存不足 | 增大 Maven 内存：MAVEN_OPTS="-Xmx1024m" |

## 四、MySQL 问题

### 4.1 MySQL 安装与启动验证

```bash
# 1. 检查 MySQL 服务是否运行（Windows）
sc query MySQL80

# 2. 登录 MySQL
mysql -u root -p
# 输入密码后进入 MySQL 命令行

# 3. 查看 MySQL 版本
SELECT VERSION();

# 4. 查看所有数据库
SHOW DATABASES;

# 5. 退出 MySQL
EXIT;
```

### 4.2 连接失败排查

```
连接失败排查流程：
+------------------+     +------------------+     +------------------+
| MySQL 服务启动？  | --> | 用户名密码正确？  | --> | 端口可访问？     |
| sc query MySQL80 |     | mysql -u root -p |     | telnet localhost |
+------------------+     +------------------+     +------------------+
       |                        |                        |
       v                        v                        v
    未启动则启动              重置密码                检查防火墙
    net start MySQL80         ALTER USER...           开放 3306
```

```bash
# 1. 检查 MySQL 服务状态
net start | findstr MySQL

# 2. 启动 MySQL 服务
net start MySQL80

# 3. 检查端口是否监听
netstat -ano | findstr :3306

# 4. 如果忘记密码，重置 root 密码（MySQL 8.0）
# 停止 MySQL 服务
net stop MySQL80

# 以跳过权限方式启动
mysqld --console --skip-grant-tables --shared-memory

# 另开终端登录（无需密码）
mysql -u root

# 重置密码
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
EXIT;

# 正常启动 MySQL 服务
net start MySQL80
```

### 4.3 字符集配置

```sql
-- 查看当前字符集配置
SHOW VARIABLES LIKE 'character%';
SHOW VARIABLES LIKE 'collation%';

-- 预期结果（推荐配置）：
-- character_set_client     = utf8mb4
-- character_set_connection = utf8mb4
-- character_set_database   = utf8mb4
-- character_set_results    = utf8mb4
-- character_set_server     = utf8mb4
```

```ini
# my.ini 配置（Windows MySQL 配置文件，通常在 C:\ProgramData\MySQL\MySQL Server 8.0\my.ini）
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

[client]
default-character-set=utf8mb4

[mysql]
default-character-set=utf8mb4
```

### 4.4 时区配置

```sql
-- 查看当前时区
SELECT @@global.time_zone, @@session.time_zone;

-- 设置为东八区（中国标准时间）
SET GLOBAL time_zone = '+8:00';
SET SESSION time_zone = '+8:00';

-- 或在 JDBC URL 中指定时区
# application.properties:
# spring.datasource.url=jdbc:mysql://localhost:3306/tlias_db?serverTimezone=Asia/Shanghai&characterEncoding=utf-8
```

### 4.5 MySQL 常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `Communications link failure` | MySQL 未启动或网络不通 | 启动 MySQL 服务；检查防火墙 |
| `Access denied for user` | 用户名/密码错误 | 检查配置；重置密码 |
| `Unknown database` | 数据库不存在 | 先执行 `CREATE DATABASE` |
| `Table doesn't exist` | 表未创建或表名错误 | 执行初始化脚本；检查表名 |
| `Data too long for column` | 字段长度不够 | 修改字段长度或缩短数据 |
| `Duplicate entry` | 唯一约束冲突 | 检查重复数据；或删除后重建 |
| 中文显示乱码 | 字符集不一致 | 统一使用 utf8mb4 |
| 时间差 8 小时 | 时区设置错误 | JDBC URL 加 `serverTimezone=Asia/Shanghai` |

## 五、IDEA 问题

### 5.1 Lombok 插件配置

Lombok 通过注解自动生成 getter/setter/toString 等方法，减少样板代码。IDEA 必须安装 Lombok 插件才能正确识别。

```
Lombok 注解作用：
+------------------+------------------------------------------+
| @Data            | 自动生成 getter/setter/toString/equals   |
| @Getter          | 只生成 getter                          |
| @Setter          | 只生成 setter                          |
| @NoArgsConstructor | 生成无参构造                          |
| @AllArgsConstructor| 生成全参构造                          |
| @Builder         | 生成建造者模式                          |
+------------------+------------------------------------------+
```

```
安装步骤：
+----------------------------------------------------------+
| 1. File -> Settings -> Plugins                           |
| 2. 搜索 "Lombok"，安装并重启 IDEA                         |
| 3. File -> Settings -> Build -> Annotation Processors    |
| 4. 勾选 "Enable annotation processing"                   |
| 5. 应用并保存                                            |
+----------------------------------------------------------+
```

### 5.2 项目编译版本设置

```
检查步骤（必须三处一致）：
+----------------------------------------------------------+
| 1. File -> Project Structure -> Project                  |
|    SDK: JDK 17 (Version B) / JDK 1.8 (Version A)        |
|    Language level: 17 / 8                                |
|                                                          |
| 2. File -> Project Structure -> Modules                  |
|    Sources -> Language level: 17 / 8                     |
|                                                          |
| 3. File -> Settings -> Build -> Compiler -> Java Compiler|
|    Target bytecode version: 17 / 8                       |
|                                                          |
| 4. pom.xml 中确认：                                      |
|    <java.version>17</java.version>                       |
+----------------------------------------------------------+
```

### 5.3 IDEA 缓存清理

```
当 IDEA 出现以下问题时，尝试清理缓存：
- 代码报错但编译通过
- 类找不到但文件存在
- 依赖已下载但 import 报错
- 运行时行为与代码不一致

清理步骤：
+----------------------------------------------------------+
| File -> Invalidate Caches...                             |
| 勾选：                                                   |
|   [x] Clear file system cache and Local History          |
|   [x] Clear VCS Log caches and indexes                   |
|   [x] Clear downloaded shared indexes                    |
| 点击 Invalidate and Restart                              |
+----------------------------------------------------------+
```

### 5.4 IDEA 常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `cannot find symbol` | 类未编译或 Lombok 未启用 | Build -> Rebuild Project；启用 Annotation Processing |
| `java: 无效的源发行版` | 编译版本与 JDK 不匹配 | 统一 Project/Module/Compiler 版本 |
| 依赖下载后仍报错 | IDEA 缓存未更新 | File -> Invalidate Caches |
| 运行按钮灰色 | 未识别为 Maven 项目 | 右键 pom.xml -> Add as Maven Project |
| 控制台中文乱码 | IDEA 编码设置问题 | Help -> Edit Custom VM Options 加 `-Dfile.encoding=UTF-8` |
| 断点不生效 | 编译优化导致 | 关闭编译优化或使用 debug 模式启动 |

## 六、Vue 前端环境问题

### 6.1 Node.js 版本管理

```bash
# 1. 查看当前 Node 版本
node -v

# Version A (Vue2): 推荐 Node 14.x ~ 16.x
# Version B (Vue3): 推荐 Node 18.x ~ 20.x

# 2. 使用 nvm 管理多版本（Windows 推荐 nvm-windows）
# 安装 nvm-windows: https://github.com/coreybutler/nvm-windows

# 查看可用版本
nvm list

# 安装指定版本
nvm install 18.20.0

# 切换版本
nvm use 18.20.0

# 3. 查看 npm 版本
npm -v
```

### 6.2 npm 镜像配置

```bash
# 1. 查看当前镜像源
npm config get registry

# 2. 设置为淘宝镜像（推荐）
npm config set registry https://registry.npmmirror.com

# 3. 或使用 nrm 管理多个镜像源
npm install -g nrm
nrm ls                  # 列出可用镜像
nrm use taobao          # 切换到淘宝镜像
nrm use npm             # 切换回官方镜像

# 4. 恢复官方镜像
npm config set registry https://registry.npmjs.org/
```

### 6.3 依赖安装与清理

```bash
# 1. 安装项目依赖
npm install
# 或简写
npm i

# 2. 如果安装失败，删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install

# Windows PowerShell:
Remove-Item -Recurse -Force node_modules
Remove-Item package-lock.json
npm install

# 3. 使用 cnpm 加速（备选方案）
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install

# 4. 查看依赖树
npm ls

# 5. 检查过期依赖
npm outdated
```

### 6.4 端口占用问题

```bash
# 1. 查找占用 8080 端口的进程（Vue 默认端口）
# Windows:
netstat -ano | findstr :8080

# 2. 结束进程
# Windows:
taskkill /PID <PID> /F

# 3. 或在 Vue 项目中修改端口
# vue.config.js:
module.exports = {
  devServer: {
    port: 3000  // 改为其他端口
  }
}

# 4. Vite 项目修改端口（Version B）
# vite.config.js:
export default {
  server: {
    port: 3000
  }
}
```

### 6.5 Vue 前端常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `npm ERR! code ENOENT` | node_modules 损坏 | 删除后重新 `npm install` |
| `Module not found` | 依赖未安装或路径错误 | `npm install`；检查 import 路径 |
| `Cannot find module 'vue'` | Vue 未安装 | `npm install vue` |
| `Syntax Error: Unexpected token` | 语法不支持 | 检查 Node 版本；检查 babel 配置 |
| 页面空白无报错 | 路由模式或 publicPath 问题 | 检查 vue-router mode；检查 publicPath |
| 热更新不生效 | webpack 缓存问题 | 重启 devServer；检查 watchOptions |
| `CORS policy` 报错 | 跨域问题 | 配置 vue.config.js proxy |

## 七、Docker 问题

### 7.1 Docker Desktop 安装与 WSL2

```
Docker Desktop + WSL2 架构：
+------------------+      +------------------+      +------------------+
|   Windows 主机    | <->  |   WSL2 虚拟机    | <->  |   Docker 引擎    |
|   (IDEA/浏览器)   |      |   (Linux 内核)   |      |   (容器运行时)   |
+------------------+      +------------------+      +------------------+

WSL2 是 Windows 的 Linux 子系统，Docker Desktop 依赖它运行 Linux 容器。
```

```powershell
# 1. 检查 WSL 状态
wsl --status

# 2. 如果未安装，执行
wsl --install

# 3. 设置 WSL2 为默认版本
wsl --set-default-version 2

# 4. 查看 WSL 发行版
wsl -l -v

# 5. 重启 WSL
wsl --shutdown
```

### 7.2 Docker 常用命令

```bash
# 1. 查看运行中的容器
docker ps

# 2. 查看所有容器（含已停止）
docker ps -a

# 3. 查看容器日志
docker logs <容器名或ID>

# 4. 实时查看日志
docker logs -f <容器名或ID>

# 5. 进入容器内部
docker exec -it <容器名> /bin/bash

# 6. 停止容器
docker stop <容器名>

# 7. 删除容器
docker rm <容器名>

# 8. 删除镜像
docker rmi <镜像名>

# 9. 清理无用资源
docker system prune
```

### 7.3 Docker 容器启动后立即退出

```bash
# 1. 查看容器日志定位原因
docker logs <容器名>

# 2. 常见原因及解决：
#    - 启动命令错误：检查 Dockerfile 或 docker-compose.yml 中的 command
#    - 端口冲突：修改映射端口
#    - 依赖服务未启动：检查数据库等服务是否先启动
#    - 配置文件错误：检查挂载的配置文件路径和内容

# 3. 以交互模式启动容器（调试）
docker run -it --entrypoint /bin/bash <镜像名>
```

### 7.4 数据卷权限问题

```bash
# Windows 下 Docker Desktop 已自动处理权限映射
# Linux/Mac 下可能需要手动设置权限

# 查看数据卷挂载情况
docker inspect <容器名> | grep -A 10 Mounts

# MySQL 容器数据目录权限问题（Linux）
chmod 777 /宿主机/数据目录

# 或在 docker-compose.yml 中指定用户
services:
  mysql:
    image: mysql:8.0
    user: "1000:1000"  # 与宿主机用户一致
```

### 7.5 Docker 常见问题

| 错误现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| `Docker Desktop starting...` 卡住 | WSL2 未启动或损坏 | `wsl --shutdown` 后重启 Docker Desktop |
| `port is already allocated` | 端口被占用 | 修改 docker-compose.yml 端口映射 |
| `no such file or directory` | 挂载路径不存在 | 创建宿主机目录；检查路径拼写 |
| 容器内无法访问外网 | DNS 配置问题 | Docker Desktop 设置中修改 DNS |
| 镜像拉取失败 | 网络问题 | 配置国内镜像加速器（阿里云/网易云） |
| 磁盘空间不足 | Docker 占用过多 | `docker system prune` 清理 |

## 八、网络问题

### 8.1 跨域问题（CORS）

```
跨域原理：
+----------------------------------------------------------+
| 浏览器安全策略：同源策略（Same-Origin Policy）             |
|                                                          |
| 同源 = 协议 + 域名 + 端口 完全相同                        |
| 例：http://localhost:8080 和 http://localhost:3000        |
|     端口不同 => 跨域！                                    |
+----------------------------------------------------------+
```

**开发环境解决方案（Vue 代理）**：

```javascript
// vue.config.js (Version A - Vue CLI)
module.exports = {
  devServer: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',  // 后端地址
        changeOrigin: true,
        pathRewrite: { '^/api': '' }      // 去掉 /api 前缀
      }
    }
  }
}

// vite.config.js (Version B - Vite)
export default defineConfig({
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '')
      }
    }
  }
})
```

**生产环境解决方案（后端 CORS）**：

```java
// SpringBoot 全局 CORS 配置
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
            .allowedOrigins("*")           // 允许所有来源（生产环境应指定具体域名）
            .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
            .allowedHeaders("*")
            .maxAge(3600);                 // 预检请求缓存时间
    }
}
```

### 8.2 防火墙问题

```powershell
# Windows 防火墙放行端口
# 1. 查看防火墙状态
netsh advfirewall show currentprofile

# 2. 放行 3306 端口（MySQL）
netsh advfirewall firewall add rule name="MySQL" dir=in action=allow protocol=tcp localport=3306

# 3. 放行 8080 端口（SpringBoot）
netsh advfirewall firewall add rule name="SpringBoot" dir=in action=allow protocol=tcp localport=8080

# 4. 关闭防火墙（仅开发环境，不推荐）
netsh advfirewall set allprofiles state off
```
