## 二、核心知识点

### 2.1 为什么需要工程化 —— 从"手写 HTML"到"现代化开发"

#### 是什么

前端工程化，指的是用工具链和规范化流程来管理前端项目的开发、构建和部署。Vue CLI（Command Line Interface）就是 Vue 官方提供的工程化脚手架工具。

#### 为什么需要它

想象你盖房子：

- **直接写 HTML/CSS/JS** = 手工搬砖、和水泥、砌墙，每块砖都要自己烧
- **工程化开发** = 工厂预制构件、塔吊吊装、标准化施工，效率高且质量稳定

直接写 HTML 文件会遇到这些问题：

| 问题 | 具体表现 |
|------|----------|
| 没有模块化 | 所有代码写在一个文件里，几千行代码难以维护 |
| 没有组件复用 | 同样的按钮、表格在每个页面复制粘贴 |
| 没有构建优化 | 代码没压缩，图片没优化，页面加载慢 |
| 没有语法检查 | 写错一个变量名，要到运行时才发现 |
| 没有热更新 | 改一行代码要手动刷新整个页面 |

Vue CLI 工程化解决了以上所有问题：

```
┌─────────────────────────────────────────────────────────────┐
│                    Vue CLI 工程化能力                         │
├─────────────────────────────────────────────────────────────┤
│  模块化开发  │  组件化复用  │  代码压缩  │  语法检查(ESLint)  │
│  热更新(HMR) │  CSS 预编译  │  资源优化  │  开发服务器代理      │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Vue CLI 安装与项目创建

#### 步骤 1：安装 Node.js

Vue CLI 依赖 Node.js 环境。先确认是否已安装：

```bash
# 打开终端（Windows 按 Win+R，输入 cmd 回车）
# 检查 Node.js 版本，要求 >= 14.0
node -v

# 检查 npm 版本
npm -v
```

如果提示"不是内部或外部命令"，说明还没安装，去 https://nodejs.org 下载 LTS 版本安装包。

#### 步骤 2：安装 Vue CLI

```bash
# -g 表示全局安装，安装后可以在任意目录使用 vue 命令
npm install -g @vue/cli

# 安装完成后，验证版本
vue --version
```

#### 步骤 3：创建项目

```bash
# 进入你想放项目的目录，比如 D 盘
D:

# 创建项目，tlias-frontend 是项目名称，可以自定义
vue create tlias-frontend
```

执行后会进入交互式配置界面，按以下步骤选择：

### 2.4 Vue 单文件组件（.vue）

#### 是什么

`.vue` 文件是 Vue 的核心创新，它将一个组件的 HTML 模板、JavaScript 逻辑、CSS 样式封装在一个文件中。

#### 为什么需要它

传统开发中，HTML、CSS、JS 写在三个不同文件里，修改一个功能要同时打开三个文件来回切换。Vue 单文件组件把相关的代码放在一起，维护更方便。

#### 结构解析

```vue
<!-- 文件名：HelloWorld.vue -->

<!-- ========== 第 1 部分：template（HTML 模板）========== -->
<!-- template 里写页面的 HTML 结构，只能有一个根元素 -->
<template>
  <!-- div 是唯一的根元素，所有内容都包在里面 -->
  <div class="hello">
    <!-- {{ }} 是 Vue 的插值表达式，显示 data 中的数据 -->
    <h1>{{ message }}</h1>
    <!-- @click 是 v-on:click 的简写，绑定点击事件 -->
    <el-button type="primary" @click="changeMessage">点击修改文字</el-button>
  </div>
</template>

<!-- ========== 第 2 部分：script（JavaScript 逻辑）========== -->
<script>
// export default 导出这个组件的配置对象
export default {
  // name: 组件名称，调试用，可选
  name: 'HelloWorld',

  // data: 定义响应式数据
  // 必须写成函数形式，返回一个对象
  data() {
    return {
      // message 是响应式数据，修改它页面会自动更新
      message: 'Hello Vue2'
    }
  },

  // methods: 定义方法
  methods: {
    // changeMessage 是一个方法，点击按钮时调用
    changeMessage() {
      // this 指向当前组件实例
      // 修改 message，页面会自动重新渲染
      this.message = 'Hello World'
    }
  },

  // created: 生命周期钩子，组件创建完成后执行
  created() {
    // 适合在这里发送初始化请求
    console.log('组件创建完成，可以发请求了')
  },

  // mounted: 生命周期钩子，DOM 挂载完成后执行
  mounted() {
    // 适合在这里操作 DOM
    console.log('DOM 挂载完成，可以操作页面元素了')
  }
}
</script>

<!-- ========== 第 3 部分：style（CSS 样式）========== -->
<!-- scoped 表示样式只作用于当前组件，不会影响其他组件 -->
<style scoped>
.hello {
  /* 文本居中 */
  text-align: center;
  /* 上边距 60 像素 */
  margin-top: 60px;
}

h1 {
  /* 文字颜色 */
  color: #409EFF;
}
</style>
```

#### 生命周期钩子图解

```
组件创建流程（从上到下执行）：

    创建阶段
       │
       ▼
   ┌─────────┐
   │ before  │  <-- 实例初始化之前（很少用）
   │ Create  │
   └────┬────┘
        │
        ▼
   ┌─────────┐
   │ created │  <-- 实例创建完成，data 和 methods 已可用
   │         │      适合：发送初始化请求
   └────┬────┘
        │
        ▼
    挂载阶段
       │
       ▼
   ┌─────────┐
   │ before  │  <-- DOM 挂载之前（很少用）
   │ Mount   │
   └────┬────┘
        │
        ▼
   ┌─────────┐
   │ mounted │  <-- DOM 挂载完成，可以操作页面元素
   │         │      适合：初始化图表、绑定第三方事件
   └────┬────┘
        │
        ▼
    更新阶段（数据变化时触发）
       │
       ▼
   ┌─────────┐     ┌─────────┐
   │ before  │────▶│ updated │  <-- 数据更新后，DOM 已重新渲染
   │ Update  │     │         │
   └─────────┘     └────┬────┘
                        │
                        ▼
                   ┌─────────┐
                   │ before  │  <-- 组件销毁之前
                   │Destroy  │      适合：清除定时器、解绑事件
                   └────┬────┘
                        │
                        ▼
                   ┌─────────┐
                   │destroyed│  <-- 组件已销毁
                   │         │
                   └─────────┘
```

### 2.5 ElementUI 引入与按需加载

#### 是什么

ElementUI 是一套基于 Vue 2.0 的桌面端组件库，提供了按钮、表格、表单、弹窗等常用组件，让我们不用从零写 CSS 就能做出漂亮的界面。

#### 为什么需要它

想象你要做一个表格：
- 不用 ElementUI：自己写 HTML table + CSS 美化 + JS 排序 + JS 分页 = 至少 200 行代码
- 用 ElementUI：`<el-table :data="list">` = 5 行代码搞定

#### 完整引入方案（适合初学者，简单直接）

```bash
# 在项目目录下执行，安装 ElementUI
npm install element-ui -S
```

```javascript
// src/main.js
// 第 1 步：引入 Vue
import Vue from 'vue'

// 第 2 步：引入 ElementUI 组件库
import ElementUI from 'element-ui'

// 第 3 步：引入 ElementUI 的样式文件（必须引入，否则组件没有样式）
import 'element-ui/lib/theme-chalk/index.css'

// 第 4 步：引入根组件
import App from './App.vue'

// 第 5 步：引入路由配置
import router from './router'

// 第 6 步：使用 ElementUI（注册所有组件）
Vue.use(ElementUI)

// 第 7 步：创建 Vue 实例
new Vue({
  router,        // 注入路由
  render: h => h(App)  // 渲染根组件
}).$mount('#app')  // 挂载到 public/index.html 的 #app 元素上
```

#### 按需加载方案（适合生产环境，减少打包体积）

完整引入会把所有组件打包进去，项目体积较大。按需加载只打包用到的组件：

```bash
# 安装按需加载插件
npm install babel-plugin-component -D
```

```javascript
// babel.config.js 修改配置
module.exports = {
  presets: ['@vue/cli-plugin-babel/preset'],
  plugins: [
    [
      'component',
      {
        libraryName: 'element-ui',
        styleLibraryName: 'theme-chalk'
      }
    ]
  ]
}
```

```javascript
// src/main.js 按需引入
import Vue from 'vue'
import { Button, Input, Table, TableColumn, Dialog, Form, FormItem } from 'element-ui'
import App from './App.vue'
import router from './router'

// 逐个注册用到的组件
Vue.use(Button)
Vue.use(Input)
Vue.use(Table)
Vue.use(TableColumn)
Vue.use(Dialog)
Vue.use(Form)
Vue.use(FormItem)

new Vue({
  router,
  render: h => h(App)
}).$mount('#app')
```

**两种方案对比**：

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 完整引入 | 简单，不用逐个引入 | 打包体积大（约 200KB+） | 学习阶段、小型项目 |
| 按需加载 | 打包体积小 | 配置稍复杂，要逐个引入 | 生产环境、大型项目 |

### 2.6 Vuex 状态管理基础

#### 是什么

Vuex 是 Vue 官方的状态管理库，用于在多个组件之间共享数据。

#### 为什么需要它

想象一个场景：
- 登录后，用户信息要在导航栏显示用户名
- 同时侧边栏要根据用户角色显示不同菜单
- 还要在页面右上角显示用户头像

如果没有 Vuex，你需要：
1. 在登录组件存数据到 localStorage
2. 在导航栏组件从 localStorage 读取
3. 在侧边栏组件从 localStorage 读取
4. 在头像组件从 localStorage 读取

有了 Vuex，所有组件共享一份数据，一处修改，处处更新：

```
┌─────────────────────────────────────────────┐
│                  Vuex Store                   │
│  ┌─────────────────────────────────────┐    │
│  │  State（数据仓库）                    │    │
│  │  { token: 'xxx', userInfo: {...} }  │    │
│  └─────────────────────────────────────┘    │
│           ▲                    │             │
│           │ 读取               │ 修改        │
│     ┌─────┴─────┐        ┌─────┴─────┐      │
│     │  导航栏    │        │  Mutations │      │
│     │  侧边栏    │        │  Actions   │      │
│     │  头像组件  │        └───────────┘      │
│     └───────────┘                             │
└─────────────────────────────────────────────┘
```

#### Vuex 核心概念

```javascript
// src/store/index.js
import Vue from 'vue'
import Vuex from 'vuex'

// 第 1 步：让 Vue 使用 Vuex 插件
Vue.use(Vuex)

// 第 2 步：创建 Store 实例
export default new Vuex.Store({
  // ==================== state ====================
  // state: 存放全局共享的数据
  // 类比：仓库的货架，存放所有货物
  state: {
    // token: 用户登录凭证，初始从 localStorage 读取
    token: localStorage.getItem('token') || '',
    // userInfo: 用户信息，初始为空对象
    userInfo: {}
  },

  // ==================== getters ====================
  // getters: 从 state 派生出的计算属性
  // 类比：仓库的统计报表，基于货架数据计算得出
  getters: {
    // isLoggedIn: 判断是否已登录
    // state 是自动传入的参数，代表当前 state 对象
    isLoggedIn(state) {
      // !! 将字符串转为布尔值：有内容返回 true，空字符串返回 false
      return !!state.token
    },
    // username: 获取用户名
    username(state) {
      return state.userInfo.name || '未登录'
    }
  },

  // ==================== mutations ====================
  // mutations: 唯一可以修改 state 的方法
  // 类比：仓库的入库/出库登记簿，所有变动都要记录
  // 必须是同步函数
  mutations: {
    // SET_TOKEN: 设置 token
    // state 是自动传入的参数，代表当前 state 对象
    // token 是调用时传入的参数
    SET_TOKEN(state, token) {
      state.token = token                    // 修改 state 中的 token
      localStorage.setItem('token', token)   // 同时持久化到 localStorage
    },
    // SET_USER_INFO: 设置用户信息
    SET_USER_INFO(state, userInfo) {
      state.userInfo = userInfo
    },
    // CLEAR_USER: 清除用户数据（退出登录用）
    CLEAR_USER(state) {
      state.token = ''
      state.userInfo = {}
      localStorage.removeItem('token')
    }
  },

  // ==================== actions ====================
  // actions: 存放异步操作和业务逻辑
  // 类比：仓库的调度员，负责协调多个操作
  // 可以包含异步代码（如发送请求）
  actions: {
    // login: 登录操作
    // context 是上下文对象，包含 commit、state、getters 等
    // credentials 是调用时传入的登录参数（用户名和密码）
    login(context, credentials) {
      // 返回 Promise，让调用方可以用 .then() 处理结果
      return new Promise((resolve, reject) => {
        // 模拟发送登录请求（实际项目中用 axios 发送真实请求）
        setTimeout(() => {
          const mockToken = 'mock-token-' + Date.now()
          // commit: 调用 mutation 修改 state
          context.commit('SET_TOKEN', mockToken)
          context.commit('SET_USER_INFO', { name: credentials.username })
          resolve(mockToken)
        }, 500)
      })
    },
    // logout: 退出登录
    logout(context) {
      context.commit('CLEAR_USER')
    }
  }
})
```

#### 在组件中使用 Vuex

```vue
<template>
  <div>
    <!-- 从 getters 读取计算属性 -->
    <p>登录状态：{{ isLoggedIn ? '已登录' : '未登录' }}</p>
    <p>用户名：{{ username }}</p>

    <!-- 登录表单 -->
    <el-input v-model="form.username" placeholder="用户名"></el-input>
    <el-input v-model="form.password" type="password" placeholder="密码"></el-input>
    <el-button type="primary" @click="handleLogin">登录</el-button>
    <el-button @click="handleLogout">退出</el-button>
  </div>
</template>

<script>
// 辅助函数：简化 mapState、mapGetters、mapMutations、mapActions 的写法
import { mapState, mapGetters, mapMutations, mapActions } from 'vuex'

export default {
  data() {
    return {
      form: {
        username: '',
        password: ''
      }
    }
  },

  computed: {
    // mapState: 将 state 中的数据映射为组件的计算属性
    // 这样可以直接用 this.token 访问
    ...mapState(['token', 'userInfo']),

    // mapGetters: 将 getters 映射为组件的计算属性
    // 这样可以直接用 this.isLoggedIn 和 this.username
    ...mapGetters(['isLoggedIn', 'username'])
  },

  methods: {
    // mapMutations: 将 mutations 映射为组件的方法
    ...mapMutations(['SET_TOKEN']),

    // mapActions: 将 actions 映射为组件的方法
    ...mapActions(['login', 'logout']),

    handleLogin() {
      // 调用 action（已经通过 mapActions 映射）
      this.login(this.form).then(() => {
        this.$message.success('登录成功')
      })
    },

    handleLogout() {
      // 调用 action
      this.logout()
      this.$message.success('已退出登录')
    }
  }
}
</script>
```

### 2.7 Axios 封装（请求拦截器 + 响应拦截器）

#### 是什么

Axios 是一个基于 Promise 的 HTTP 客户端，用于发送 AJAX 请求。"封装"指的是在 Axios 基础上添加统一的处理逻辑，避免在每个页面重复写同样的代码。

#### 为什么需要封装

不封装的写法（每个页面都要写这些）：

```javascript
// 页面 A：获取部门列表
axios.get('/api/depts', {
  headers: { token: localStorage.getItem('token') }
}).then(res => {
  if (res.data.code === 0) {
    this.$message.error(res.data.msg)
    return
  }
  this.list = res.data.data
})

// 页面 B：获取员工列表
axios.get('/api/emps', {
  headers: { token: localStorage.getItem('token') }
}).then(res => {
  if (res.data.code === 0) {
    this.$message.error(res.data.msg)
    return
  }
  this.list = res.data.data
})
```

封装后的写法（简洁优雅）：

```javascript
// 页面 A
this.$request.get('/depts').then(data => {
  this.list = data
})

// 页面 B
this.$request.get('/emps').then(data => {
  this.list = data
})
```

#### 完整封装代码

```javascript
// src/utils/request.js

// 第 1 步：引入 axios
import axios from 'axios'

// 第 2 步：引入 ElementUI 的消息提示组件
import { Message } from 'element-ui'

// 第 3 步：引入路由实例（用于跳转登录页）
import router from '../router'

// 第 4 步：创建 axios 实例
// create 方法可以自定义配置，比如基础 URL、超时时间等
const request = axios.create({
  // baseURL: 所有请求的 URL 前缀
  // 开发时通过代理转发到后端，生产环境可配置为真实域名
  baseURL: '/api',
  // timeout: 请求超时时间（毫秒）
  // 超过 10 秒没响应就报错
  timeout: 10000
})

// ==================== 请求拦截器 ====================
// 作用：在请求发送前做一些统一处理
// 类比：寄快递前的打包检查

request.interceptors.request.use(
  // config: 请求配置对象，包含 url、method、headers、data 等
  config => {
    // 第 1 步：从 localStorage 获取 token
    const token = localStorage.getItem('token')

    // 第 2 步：如果有 token，添加到请求头
    // 后端通过 headers.token 获取用户身份
    if (token) {
      config.headers.token = token
    }

    // 第 3 步：返回修改后的配置（必须返回，否则请求不会发送）
    return config
  },
  // error: 请求发送失败时的处理
  error => {
    console.error('请求发送失败：', error)
    return Promise.reject(error)
  }
)

// ==================== 响应拦截器 ====================
// 作用：在收到响应后做一些统一处理
// 类比：收到快递后的验货处理

request.interceptors.response.use(
  // 第 1 个函数：处理成功响应（HTTP 状态码 2xx）
  response => {
    // response.data 是后端返回的 JSON 数据
    // 假设后端统一返回格式：{ code: 1, msg: '操作成功', data: {...} }
    const res = response.data

    // 判断业务状态码
    if (res.code === 0) {
      // code === 0 表示业务失败（注意：这是后端约定的，不是 HTTP 错误）
      // 显示错误提示
      Message.error(res.msg || '请求失败')

      // 如果是未登录错误，清除 token 并跳转登录页
      if (res.msg === 'NOT_LOGIN') {
        localStorage.removeItem('token')
        router.push('/login')
      }

      // 返回一个被拒绝的 Promise，让调用方进入 catch 分支
      return Promise.reject(new Error(res.msg))
    }

    // code === 1 表示业务成功，直接返回 data 部分
    // 这样调用方可以直接拿到数据，不用每次都 res.data.data
    return res.data
  },
  // 第 2 个函数：处理错误响应（HTTP 状态码 4xx、5xx）
  error => {
    // 从 error 对象中提取错误信息
    const message = error.message || '网络错误'

    // 显示错误提示
    Message.error(message)

    // 返回被拒绝的 Promise
    return Promise.reject(error)
  }
)

// 第 5 步：导出封装好的请求实例
export default request
```

#### 挂载到 Vue 原型

```javascript
// src/main.js
import Vue from 'vue'
import ElementUI from 'element-ui'
import 'element-ui/lib/theme-chalk/index.css'
import App from './App.vue'
import router from './router'
import store from './store'

// 引入封装好的请求工具
import request from './utils/request'

// 将 request 挂载到 Vue 原型上
// 这样在所有组件中都可以通过 this.$request 使用
Vue.prototype.$request = request

Vue.use(ElementUI)

new Vue({
  router,
  store,
  render: h => h(App)
}).$mount('#app')
```

#### 在组件中使用封装后的请求

```vue
<script>
export default {
  data() {
    return {
      deptList: []
    }
  },
  created() {
    // 发送 GET 请求
    // 不需要手动加 token，拦截器会自动处理
    // 不需要判断 code，拦截器已经处理过了
    // 直接拿到 data 部分
    this.$request.get('/depts').then(data => {
      this.deptList = data
    }).catch(err => {
      // 只有真正出错时才进这里（网络错误、业务失败）
      console.log('请求失败：', err)
    })
  },
  methods: {
    // 发送 POST 请求
    addDept() {
      this.$request.post('/depts', { name: '研发部' }).then(() => {
        this.$message.success('新增成功')
      })
    },
    // 发送 PUT 请求
    updateDept() {
      this.$request.put('/depts', { id: 1, name: '技术部' }).then(() => {
        this.$message.success('修改成功')
      })
    },
    // 发送 DELETE 请求
    deleteDept(id) {
      this.$request.delete(`/depts/${id}`).then(() => {
        this.$message.success('删除成功')
      })
    }
  }
}
</script>
```

### 2.8 开发环境代理配置

#### 是什么

开发时前端运行在 `localhost:8080`，后端运行在 `localhost:8080`，端口不同会产生跨域问题。Vue CLI 提供了代理配置，将前端的请求转发到后端。

#### 配置方法

```javascript
// vue.config.js（项目根目录下新建这个文件）
module.exports = {
  // devServer: 开发服务器配置
  devServer: {
    // port: 前端开发服务器端口
    port: 3000,
    // proxy: 代理配置
    proxy: {
      // '/api': 匹配以 /api 开头的请求
      '/api': {
        // target: 目标服务器地址（后端地址）
        target: 'http://localhost:8080',
        // changeOrigin: 是否改变请求源头（解决跨域）
        changeOrigin: true,
        // pathRewrite: 路径重写
        // 将 /api/depts 重写为 /depts（去掉 /api 前缀）
        pathRewrite: {
          '^/api': ''
        }
      }
    }
  }
}
```

**代理转发示意图**：

```
浏览器请求流程：

  浏览器
    │  请求：http://localhost:3000/api/depts
    │
    ▼
  Vue 开发服务器 (localhost:3000)
    │  匹配到 /api 前缀，转发到 target
    │
    ▼
  后端服务器 (localhost:8080)
    │  收到请求：/depts（/api 已被重写去掉）
    │
    ▼
  返回 JSON 数据
```

## 四、常见错误排查

### 4.1 依赖配置问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| `vue: command not found` | Vue CLI 未全局安装 | 执行 `npm install -g @vue/cli` |
| `Cannot find module 'element-ui'` | ElementUI 未安装 | 执行 `npm install element-ui -S` |
| `Cannot find module 'axios'` | Axios 未安装 | 执行 `npm install axios -S` |
| `Module not found: Error: Can't resolve 'element-ui/lib/theme-chalk/index.css'` | CSS 路径写错 | 检查是否为 `element-ui/lib/theme-chalk/index.css` |
| `npm ERR! code EACCES` | npm 权限不足（Mac/Linux） | 不要用 sudo，改用 nvm 管理 Node 版本 |

### 4.2 参数请求问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 请求 404，提示 `/api/depts` 找不到 | 代理配置未生效 | 检查 `vue.config.js` 是否在根目录，重启服务 |
| 后端收不到 token | 请求头字段名不匹配 | 确认后端读取的是 `token` 还是 `Authorization` |
| 请求超时 | 后端服务未启动或网络不通 | 确认后端运行在 `localhost:8080` |
| 跨域错误 `CORS` | 代理配置错误 | 检查 `changeOrigin: true` 是否配置 |

### 4.3 代码逻辑问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 修改数据后页面不更新 | 直接修改数组索引或对象属性 | 使用 `Vue.set()` 或替换整个对象/数组 |
| `this.$request is not a function` | 未挂载到 Vue 原型 | 检查 `main.js` 中 `Vue.prototype.$request = request` |
| Vuex 数据修改后组件没反应 | 直接修改 state，没走 mutation | 必须用 `commit` 调用 mutation |
| 组件样式影响了其他组件 | 没加 `scoped` | 在 `<style>` 标签上加 `scoped` |
| `Cannot read property 'xxx' of undefined` | 异步数据未加载完就渲染 | 用 `v-if` 判断数据存在后再渲染 |

### 4.4 性能安全问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 打包后的文件很大（> 2MB） | ElementUI 完整引入 | 改用按需加载，或配置 CDN |
| 页面首次加载慢 | 没做代码分割 | 路由组件用 `() => import()` 懒加载 |
| token 存在 localStorage 中，有 XSS 风险 | 存储位置选择不当 | 了解 XSS 攻击原理，配合后端做 token 过期校验 |
| 生产环境接口地址写死在代码里 | 硬编码 API 地址 | 使用环境变量 `process.env.VUE_APP_API_URL` |
| 源码泄露（能看到 .vue 文件） | 没正确构建生产包 | 用 `npm run build` 构建，不是直接复制 src |
