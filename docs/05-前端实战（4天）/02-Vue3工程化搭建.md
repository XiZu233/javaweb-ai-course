## 二、核心知识点

### 2.1 Vue3 相比 Vue2 的核心变化

#### 是什么

Vue3 是 Vue 框架的重大版本升级，于 2020 年 9 月发布。它在保持 Vue 易用性的同时，带来了更好的性能、更优的 TypeScript 支持和全新的 Composition API。

#### 为什么需要升级

想象你经营一家餐厅：
- **Vue2** = 传统中餐厨房，每个厨师负责一道菜，分工明确但协作复杂
- **Vue3** = 现代化中央厨房，按食材和工序组织，更灵活、更高效

Vue3 的核心改进一览：

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Vue3 核心改进全景图                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   性能提升                    开发体验                    架构优化   │
│   ─────────                  ─────────                  ─────────  │
│   │ 打包体积减少 41% │       │ TypeScript 原生支持 │    │ 模块化设计 │
│   │ 内存占用减少 54% │       │ 更好的代码提示      │    │ 树摇优化   │
│   │ 渲染速度提升 55% │       │ 更清晰的错误提示    │    │ 组合式 API │
│   │ 更新速度提升 133%│       │ 源码更易读         │    │ 自定义渲染器│
│   └─────────────────┘       └─────────────────┘    └───────────┘  │
│                                                                     │
│   响应式系统升级                                                    │
│   ────────────────                                                  │
│   Vue2: Object.defineProperty  （只能监听已有属性，新增属性需 Vue.set）│
│   Vue3: Proxy                   （监听整个对象，新增属性自动响应）    │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

#### Vue2 vs Vue3 详细对比

| 特性 | Vue2 | Vue3 | 影响 |
|------|------|------|------|
| 响应式原理 | `Object.defineProperty` | `Proxy` | Vue3 可以监听新增属性和数组索引变化 |
| 新增属性响应 | 需用 `Vue.set(obj, key, val)` | 直接赋值即可 | Vue3 代码更简洁 |
| 根节点数量 | 只能有一个根元素 | 可以有多个根元素 | Vue3 模板更灵活 |
| 生命周期命名 | `created` / `beforeDestroy` | `setup` 中 `onMounted` / `onUnmounted` | Vue3 命名更语义化 |
| 逻辑复用 | Mixins（容易命名冲突） | Composition API（灵活组合） | Vue3 复用更清晰 |
| 性能 | 好 | 更好（静态提升、PatchFlag） | Vue3 更快更省内存 |
| TypeScript | 支持一般 | 原生支持，类型推断更好 | Vue3 大型项目更稳 |
| 创建实例 | `new Vue({...})` | `createApp(...)` | Vue3 API 更现代 |

### 2.2 Vite vs Webpack —— 为什么 Vue3 推荐 Vite

#### 是什么

Vite 是 Vue 作者尤雨溪开发的新一代前端构建工具，意为"快速的"（法语）。Webpack 是目前最主流的打包工具，但配置复杂、启动慢。

#### 为什么 Vue3 推荐 Vite

想象你去餐厅吃饭：
- **Webpack** = 传统餐厅，你点完菜后厨师才开始买菜、洗菜、切菜、炒菜，等很久才上菜（启动慢）
- **Vite** = 快餐店，大部分食材已经预处理好了，你点完直接组装，秒上菜（启动快）

**核心差异对比**：

| 对比项 | Webpack | Vite |
|--------|---------|------|
| 开发启动 | 先打包所有模块，再启动（冷启动 10-30 秒） | 利用浏览器原生 ESM，秒级启动（< 1 秒） |
| 热更新（HMR） | 修改后重新编译整个模块链（慢） | 只更新修改的模块（快） |
| 生产构建 | 自己实现打包逻辑 | 用 Rollup 打包，输出更优 |
| 配置复杂度 | 复杂，需理解 loader、plugin 等概念 | 简单，大部分场景零配置 |
| 原理 | 将所有模块打包成一个/多个 bundle | 开发时直接让浏览器请求原文件 |

**Vite 开发时的工作原理**：

```
Webpack 模式（慢）：

  浏览器 ──▶ 请求页面 ──▶ 服务器
                              │
                              ▼
                         打包所有模块
                         （分析依赖、转译、合并）
                              │
                              ▼
                         生成 bundle.js
                              │
                              ▼
  浏览器 ◀── 返回 bundle.js ◀──┘
  （文件很大，加载慢）

Vite 模式（快）：

  浏览器 ──▶ 请求页面 ──▶ 服务器
                              │
                              ▼
                         直接返回 HTML
                         （不做打包！）
                              │
                              ▼
  浏览器 ◀── 返回 HTML ◀─────┘
    │
    ▼ 浏览器解析到 import 语句
    │
    ├──▶ 请求 /src/main.ts ──▶ 服务器即时转译 ──▶ 返回
    ├──▶ 请求 /src/App.vue ──▶ 服务器即时编译 ──▶ 返回
    └──▶ 请求 /node_modules/vue ──▶ 服务器预构建 ──▶ 返回

  （按需加载，改哪个文件只更新哪个）
```

### 2.5 Composition API —— setup、ref、reactive、computed、watch

#### 是什么

Composition API（组合式 API）是 Vue3 引入的全新组织代码的方式。与 Vue2 的 Options API（按选项组织：data、methods、computed 分开写）不同，Composition API 按功能逻辑组织代码。

#### 为什么需要它

想象你要实现一个"用户搜索"功能，包含：
- 搜索关键词（数据）
- 搜索结果列表（数据）
- 加载状态（数据）
- 搜索方法（逻辑）
- 防抖处理（逻辑）

**Options API 写法（Vue2）**：

```javascript
export default {
  data() {
    return {
      keyword: '',        // 搜索关键词 ──┐
      resultList: [],     // 结果列表 ────┤ 这些相关的逻辑被拆散在不同选项里
      loading: false      // 加载状态 ────┘
    }
  },
  computed: {
    // 计算属性在这里
  },
  watch: {
    // 监听在这里
  },
  methods: {
    search() {           // 搜索方法在这里 ──┐
      // ...                              ├─ 相关逻辑分散在各处
    },                                  │
    debounce() {        // 防抖在这里 ─────┘
      // ...
    }
  }
}
```

**Composition API 写法（Vue3）**：

```typescript
// 所有"搜索功能"相关的代码都在一起，逻辑内聚
function useSearch() {
  const keyword = ref('')           // 关键词
  const resultList = ref([])        // 结果
  const loading = ref(false)        // 加载状态

  const search = async () => {     // 搜索方法
    loading.value = true
    // ...
    loading.value = false
  }

  watch(keyword, debounce(search, 300))  // 监听 + 防抖

  return { keyword, resultList, loading, search }
}

// 在组件中使用
const { keyword, resultList, loading, search } = useSearch()
```

#### script setup 语法

Vue3 推荐使用 `<script setup>` 语法，代码更简洁：

```vue
<!-- 文件名：Counter.vue -->

<template>
  <!-- Vue3 支持多个根元素，不需要包在一个 div 里 -->
  <div class="counter">
    <h2>计数器示例</h2>
    <!-- 模板中直接使用变量，不需要 this -->
    <p>当前计数：{{ count }}</p>
    <p>双倍计数：{{ doubleCount }}</p>
    <el-button type="primary" @click="increment">+1</el-button>
    <el-button @click="decrement">-1</el-button>
    <p>消息：{{ message }}</p>
    <el-button @click="changeMessage">修改消息</el-button>
  </div>
</template>

<!-- lang="ts" 表示使用 TypeScript -->
<script setup lang="ts">
// 第 1 步：从 vue 包中引入需要的 API
import { ref, reactive, computed, watch, onMounted } from 'vue'

// ==================== ref：基本类型的响应式数据 ====================
// ref 用于包装基本类型（string、number、boolean）为响应式数据
// 类比：给普通变量套上一个"响应式盒子"

// 定义一个响应式数字
// count 的类型是 Ref<number>，不是普通的 number
const count = ref(0)

// 在 script 中读取/修改 ref 的值，必须通过 .value
// 在 template 中直接使用，不需要 .value
const increment = () => {
  count.value++   // 修改 .value 才能触发响应式更新
}

const decrement = () => {
  count.value--
}

// ==================== computed：计算属性 ====================
// computed 用于根据已有响应式数据派生出新数据
// 类比：Excel 中的公式，依赖的单元格变了，公式结果自动更新
// 有缓存：依赖不变时不会重新计算

const doubleCount = computed(() => {
  // 当 count.value 变化时，doubleCount 自动重新计算
  return count.value * 2
})

// ==================== reactive：对象类型的响应式数据 ====================
// reactive 用于将对象转为响应式
// 类比：给整个对象套上一个"响应式盒子"
// 注意：reactive 只能用于对象，不能用于基本类型

// 定义一个响应式对象
const user = reactive({
  name: '张三',
  age: 25,
  address: {
    city: '北京',
    district: '海淀区'
  }
})

// 修改 reactive 对象的属性，不需要 .value
const updateUser = () => {
  user.name = '李四'           // 直接修改即可
  user.age = 30
  user.address.city = '上海'   // 嵌套属性也能响应
}

// ==================== watch：监听数据变化 ====================
// watch 用于监听响应式数据的变化，执行回调
// 类比：给数据装一个"监控摄像头"，变化时自动报警

// 监听单个 ref
watch(count, (newVal, oldVal) => {
  console.log(`count 变化了：${oldVal} → ${newVal}`)
})

// 监听 reactive 对象的某个属性（要用 getter 函数）
watch(() => user.name, (newVal, oldVal) => {
  console.log(`用户名变化了：${oldVal} → ${newVal}`)
})

// 监听多个数据源
watch([count, () => user.age], ([newCount, newAge], [oldCount, oldAge]) => {
  console.log('多个数据变化：', newCount, newAge)
})

// ==================== 生命周期钩子 ====================
// Vue3 的生命周期钩子以 "on" 开头，需要在 setup 中显式引入

onMounted(() => {
  // DOM 挂载完成，可以操作页面元素
  console.log('组件挂载完成')
})

// 其他生命周期钩子：
// onBeforeMount、onUpdated、onBeforeUpdate、onUnmounted、onBeforeUnmount

// ==================== 普通变量（非响应式）====================
// 不是所有变量都需要响应式，普通变量性能更好
const message = ref('Hello Vue3')

const changeMessage = () => {
  message.value = 'Hello World'
}
</script>

<style scoped>
.counter {
  padding: 20px;
  text-align: center;
}
p {
  font-size: 18px;
  color: #333;
}
</style>
```

#### ref vs reactive 对比

| 特性 | ref | reactive |
|------|-----|----------|
| 适用类型 | 基本类型 + 对象 | 仅限对象 |
| 访问方式 | `.value` | 直接访问属性 |
| 模板中使用 | 直接写变量名 | 直接写变量名.属性 |
| 替换整个对象 | `obj.value = newObj` | 不能替换，会丢失响应式 |
| 解构 | 解构后丢失响应式 | 用 `toRefs` 保持响应式 |
| 推荐场景 | 单个值、需要替换的对象 | 结构稳定的对象 |

**最佳实践**：

```typescript
// 推荐：基本类型用 ref
const name = ref('张三')
const age = ref(25)
const isVip = ref(false)

// 推荐：对象类型根据场景选择
// 场景 1：对象结构固定，不需要替换整个对象 ──▶ 用 reactive
const form = reactive({
  username: '',
  password: '',
  email: ''
})

// 场景 2：对象需要整体替换（如从接口获取新数据）──▶ 用 ref
const userInfo = ref({ name: '', age: 0 })
// 可以直接替换
userInfo.value = { name: '李四', age: 30 }
```

### 2.6 TypeScript 基础入门

#### 是什么

TypeScript（TS）是 JavaScript 的超集，添加了静态类型检查。代码写完后编译时就能发现类型错误，而不是等到运行时才暴露。

#### 为什么需要它

想象你在组装家具：
- **JavaScript** = 没有说明书，装错了要到用时才发现（运行时报错）
- **TypeScript** = 有详细的说明书，装错一步就提醒你（编译时报错）

#### 基础类型

```typescript
// ========== 基本类型 ==========

// string: 字符串
const userName: string = '张三'

// number: 数字（整数和小数都是 number）
const age: number = 25
const price: number = 19.99

// boolean: 布尔值
const isActive: boolean = true

// null 和 undefined
const empty: null = null
const notDefined: undefined = undefined

// any: 任意类型（尽量避免使用，会失去类型检查）
let anything: any = 'hello'
anything = 123      // 不会报错
anything = true     // 不会报错

// unknown: 未知类型（比 any 安全，使用前需要类型断言）
let unknownValue: unknown = 'hello'
// unknownValue.toUpperCase()  // 报错！需要先判断类型
if (typeof unknownValue === 'string') {
  unknownValue.toUpperCase()   // 安全了
}

// ========== 数组类型 ==========

// 方式 1：类型[]
const numbers: number[] = [1, 2, 3, 4, 5]

// 方式 2：Array<类型>（泛型写法）
const names: Array<string> = ['张三', '李四', '王五']

// ========== 对象类型 ==========

// 使用接口（interface）定义对象结构
interface User {
  id: number        // id 是数字，必填
  name: string      // name 是字符串，必填
  age?: number      // age 是数字，? 表示可选
  email?: string    // email 可选
}

// 使用接口定义变量
const user: User = {
  id: 1,
  name: '张三',
  age: 25
  // email 是可选的，可以不写
}

// ========== 函数类型 ==========

// 定义函数参数类型和返回值类型
function add(a: number, b: number): number {
  return a + b
}

// 箭头函数
const greet = (name: string): string => {
  return `你好，${name}`
}

// 没有返回值的函数（void）
function logMessage(msg: string): void {
  console.log(msg)
}

// ========== 泛型入门 ==========
// 泛型：让函数/组件支持多种类型，同时保持类型安全

// 没有泛型：只能处理一种类型
function identityString(arg: string): string {
  return arg
}

// 使用泛型：可以处理任意类型，同时保持类型推断
function identity<T>(arg: T): T {
  return arg
}

// 使用时指定类型
const result1 = identity<string>('hello')   // result1 的类型是 string
const result2 = identity<number>(123)      // result2 的类型是 number

// TypeScript 会自动推断类型，通常不需要显式指定
const result3 = identity('hello')           // 自动推断为 string
```

#### 在 Vue3 组件中使用 TypeScript

```vue
<script setup lang="ts">
import { ref } from 'vue'

// ========== 定义接口 ==========
// 接口描述了后端返回的部门数据结构
interface Dept {
  id: number           // 部门 ID
  name: string         // 部门名称
  createTime: string   // 创建时间
  updateTime: string   // 更新时间
}

// ========== 使用接口定义响应式数据 ==========
// deptList 的类型是 Ref<Dept[]>，表示"部门数组的响应式引用"
const deptList = ref<Dept[]>([])

// ========== 定义函数参数和返回值类型 ==========
// async 函数返回 Promise
const fetchDeptList = async (): Promise<void> => {
  // 模拟请求
  const res: Dept[] = [
    { id: 1, name: '研发部', createTime: '2024-01-01', updateTime: '2024-01-01' },
    { id: 2, name: '市场部', createTime: '2024-01-02', updateTime: '2024-01-02' }
  ]
  deptList.value = res
}

// ========== 定义表单接口 ==========
interface DeptForm {
  id: number | null   // ID 新增时为 null
  name: string        // 部门名称
}

const form = ref<DeptForm>({
  id: null,
  name: ''
})

// 提交表单
const submit = async (): Promise<void> => {
  if (form.value.id) {
    console.log('编辑部门：', form.value)
  } else {
    console.log('新增部门：', form.value)
  }
}
</script>
```

### 2.8 Pinia 状态管理

#### 是什么

Pinia 是 Vue3 官方推荐的状态管理库，由 Vue 核心团队成员开发。它是 Vuex 的替代品，API 更简洁，TypeScript 支持更好。

#### 为什么用 Pinia 替代 Vuex

| 对比项 | Vuex | Pinia |
|--------|------|-------|
| 语法复杂度 | 需要 mutations、actions、getters 严格区分 | 更灵活，可以直接修改 state |
| TypeScript 支持 | 需要额外配置 | 原生支持，类型推断完美 |
| 代码量 | 较多（需要定义常量、mutations 等） | 较少（一个函数搞定） |
| 模块化 | 需要 namespaced | 天然模块化，每个 store 独立 |
| 体积 | ~1KB | ~1KB（更轻量） |
| 开发者工具 | 支持 | 支持，体验更好 |

#### Pinia 核心概念

Pinia 的 API 设计非常直观：

```
Vuex 的写法                    Pinia 的写法
─────────────                 ─────────────
state                         state（就是 ref/reactive）
getters                       getters（就是 computed）
mutations                     没有了！直接修改 state
actions                       actions（就是普通函数）
```

#### 完整 Store 示例

```typescript
// src/stores/user.ts

// 第 1 步：从 pinia 引入 defineStore
import { defineStore } from 'pinia'

// 第 2 步：从 vue 引入响应式 API
import { ref, computed } from 'vue'

// 第 3 步：定义 store
// 参数 1：store 的唯一标识（ID），全局唯一
// 参数 2：工厂函数，返回 store 的对象
export const useUserStore = defineStore('user', () => {
  // ==================== state ====================
  // 用 ref 定义状态（相当于 Vuex 的 state）
  const token = ref<string>(localStorage.getItem('token') || '')
  const userInfo = ref<any>({})

  // ==================== getters ====================
  // 用 computed 定义派生状态（相当于 Vuex 的 getters）
  const isLoggedIn = computed(() => {
    // !! 将字符串转为布尔值
    return !!token.value
  })

  // 获取用户名
  const username = computed(() => {
    return userInfo.value.name || '未登录'
  })

  // ==================== actions ====================
  // 普通函数就是 actions（相当于 Vuex 的 actions + mutations）

  // 设置 token
  const setToken = (val: string) => {
    token.value = val                    // 直接修改，不需要 mutation
    localStorage.setItem('token', val)   // 持久化存储
  }

  // 设置用户信息
  const setUserInfo = (info: any) => {
    userInfo.value = info
  }

  // 退出登录
  const logout = () => {
    token.value = ''
    userInfo.value = {}
    localStorage.removeItem('token')
  }

  // ==================== 返回所有内容 ====================
  // 返回的内容才能在组件中使用
  return {
    token,
    userInfo,
    isLoggedIn,
    username,
    setToken,
    setUserInfo,
    logout
  }
})
```

#### 在组件中使用 Pinia

```vue
<script setup lang="ts">
// 第 1 步：引入 store
import { useUserStore } from '@/stores/user'

// 第 2 步：创建 store 实例
const userStore = useUserStore()

// 第 3 步：直接使用 store 中的数据和方法
// 在模板中可以直接用 userStore.token、userStore.isLoggedIn 等

const handleLogin = async () => {
  // 调用 store 的 action
  userStore.setToken('mock-token-123')
  userStore.setUserInfo({ name: '张三', role: 'admin' })
}

const handleLogout = () => {
  userStore.logout()
}
</script>

<template>
  <div>
    <p>登录状态：{{ userStore.isLoggedIn ? '已登录' : '未登录' }}</p>
    <p>用户名：{{ userStore.username }}</p>
    <el-button type="primary" @click="handleLogin">模拟登录</el-button>
    <el-button @click="handleLogout">退出登录</el-button>
  </div>
</template>
```

#### Pinia 的解构问题

```typescript
// 错误：直接解构会丢失响应式
const { token, isLoggedIn } = useUserStore()
// token 和 isLoggedIn 变成了普通值，不会随 store 更新

// 正确：使用 storeToRefs 保持响应式
import { storeToRefs } from 'pinia'

const userStore = useUserStore()
const { token, isLoggedIn } = storeToRefs(userStore)
// 现在 token 和 isLoggedIn 是响应式的 ref

// 方法不需要 storeToRefs，直接解构即可
const { setToken, logout } = userStore
```

### 2.9 Axios 封装（TypeScript 版）

```typescript
// src/utils/request.ts

// 第 1 步：引入 axios 和类型
import axios, { type AxiosInstance, type AxiosResponse } from 'axios'

// 第 2 步：引入 ElementPlus 的消息提示
import { ElMessage } from 'element-plus'

// 第 3 步：引入路由和 Pinia store
import router from '@/router'
import { useUserStore } from '@/stores/user'

// 第 4 步：定义后端响应的统一结构
interface ApiResponse<T = any> {
  code: number      // 业务状态码：1 成功，0 失败
  msg: string       // 提示消息
  data: T           // 响应数据，用泛型表示可以是任意类型
}

// 第 5 步：创建 axios 实例
const request: AxiosInstance = axios.create({
  // baseURL: 从环境变量读取，没有则使用 /api
  baseURL: import.meta.env.VITE_API_BASE_URL || '/api',
  // 请求超时时间：10 秒
  timeout: 10000
})

// ==================== 请求拦截器 ====================
request.interceptors.request.use(
  (config) => {
    // 从 Pinia store 获取 token
    const userStore = useUserStore()

    // 如果有 token，添加到请求头
    if (userStore.token) {
      // 使用 Bearer Token 格式（行业标准）
      config.headers.Authorization = 'Bearer ' + userStore.token
    }

    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)

// ==================== 响应拦截器 ====================
request.interceptors.response.use(
  // 处理成功响应（HTTP 2xx）
  (response: AxiosResponse<ApiResponse>) => {
    const res = response.data

    // 判断业务状态码
    if (res.code !== 1) {
      // 业务失败
      ElMessage.error(res.msg || '请求失败')

      // 401 未授权，清除登录状态并跳转
      if (res.code === 401) {
        const userStore = useUserStore()
        userStore.logout()
        router.push('/login')
      }

      return Promise.reject(new Error(res.msg))
    }

    // 业务成功，返回 data 部分
    return res.data
  },
  // 处理错误响应（HTTP 4xx/5xx）
  (error) => {
    const message = error.message || '网络错误'
    ElMessage.error(message)
    return Promise.reject(error)
  }
)

// 第 6 步：导出请求实例
export default request
```

## 四、常见错误排查

### 4.1 依赖配置问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| `Cannot find module 'vue'` | 依赖未安装 | 执行 `npm install` |
| `Cannot find module 'element-plus'` | ElementPlus 未安装 | 执行 `npm install element-plus` |
| `Cannot find module '@/stores/user'` | 路径别名配置错误 | 检查 `vite.config.ts` 中 `resolve.alias` |
| `TS2307: Cannot find module './App.vue'` | TS 不认识 .vue 文件 | 确保 `env.d.ts` 中有 `declare module '*.vue'` |
| `Vite 启动报错 port already in use` | 端口被占用 | 修改 `vite.config.ts` 中的 port |

### 4.2 参数请求问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 请求 404 | 代理配置未生效 | 检查 `vite.config.ts` 的 proxy 配置，注意 rewrite 语法 |
| 后端收不到 token | 请求头字段名不匹配 | 确认后端读取的是 `Authorization` 还是 `token` |
| 环境变量读取为 undefined | 前缀不对 | Vite 要求环境变量以 `VITE_` 开头 |
| 跨域错误 | 代理配置错误 | 检查 `changeOrigin: true` 是否配置 |

### 4.3 代码逻辑问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 修改 ref 后页面不更新 | 直接赋值而非 `.value` | `ref` 需要通过 `.value` 读写 |
| reactive 对象解构后丢失响应式 | 解构破坏了 Proxy 代理 | 使用 `toRefs` 或保持对象引用 |
| TypeScript 类型报错 `Type 'X' is not assignable` | 类型不匹配 | 显式指定类型，或使用 `as` 类型断言 |
| Pinia store 修改后组件没反应 | 直接解构了 state | 使用 `storeToRefs` 保持响应式 |
| `onMounted` 没执行 | 没从 vue 引入 | `import { onMounted } from 'vue'` |
| 模板中访问不到变量 | 变量没在 setup 中返回 | 确保变量在 `return` 中暴露（script setup 自动处理） |

### 4.4 性能安全问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 打包后文件很大 | 没做代码分割 | 路由组件用 `() => import()` 懒加载 |
| 页面首次加载慢 | 没配置 CDN | 生产环境用 CDN 引入 Vue、ElementPlus |
| token 存储安全问题 | 存在 XSS 风险 | 了解 XSS 原理，配合后端做 token 过期校验 |
| 类型滥用 `any` | 代码失去类型保护 | 尽量用具体类型，必要时用 `unknown` |
| 内存泄漏 | 组件卸载时没清理 | 在 `onUnmounted` 中清除定时器、事件监听 |
