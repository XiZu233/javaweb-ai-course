### 1. 登录页面

```vue
<template>
  <div class="login-container">
    <el-card class="login-box">
      <div slot="header" class="login-title">Tlias 管理系统</div>
      <el-form :model="form" :rules="rules" ref="form">
        <el-form-item prop="username">
          <el-input v-model="form.username" placeholder="用户名" prefix-icon="el-icon-user"></el-input>
        </el-form-item>
        <el-form-item prop="password">
          <el-input v-model="form.password" type="password" placeholder="密码" prefix-icon="el-icon-lock"></el-input>
        </el-form-item>
        <el-form-item>
          <el-button type="primary" style="width: 100%;" @click="login" :loading="loading">登录</el-button>
        </el-form-item>
      </el-form>
    </el-card>
  </div>
</template>

<script>
export default {
  data() {
    return {
      loading: false,
      form: { username: 'admin', password: '123456' },
      rules: {
        username: [{ required: true, message: '请输入用户名', trigger: 'blur' }],
        password: [{ required: true, message: '请输入密码', trigger: 'blur' }]
      }
    }
  },
  methods: {
    login() {
      this.$refs.form.validate(valid => {
        if (valid) {
          this.loading = true
          this.$request.post('/login', this.form).then(token => {
            localStorage.setItem('token', token)
            this.$message.success('登录成功')
            this.$router.push('/')
          }).catch(() => {}).finally(() => {
            this.loading = false
          })
        }
      })
    }
  }
}
</script>
```

### 3. 路由守卫

路由守卫用于在路由跳转前进行权限校验：

```javascript
// router/index.js
import Vue from 'vue'
import VueRouter from 'vue-router'

Vue.use(VueRouter)

const router = new VueRouter({
  mode: 'history',
  routes: [
    { path: '/login', component: () => import('../views/login/index.vue') },
    {
      path: '/',
      component: () => import('../views/layout/index.vue'),
      redirect: '/dept',
      children: [
        { path: 'dept', component: () => import('../views/dept/index.vue'), meta: { title: '部门管理' } },
        { path: 'emp', component: () => import('../views/emp/index.vue'), meta: { title: '员工管理' } }
      ]
    }
  ]
})

// 全局前置守卫
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')
  if (to.path !== '/login' && !token) {
    next('/login')   // 未登录，跳转到登录页
  } else {
    next()           // 已登录，放行
  }
})

export default router
```

### 4. 完整登录流程

```
用户打开系统
    ↓
路由守卫检查：无 Token → 跳转登录页
    ↓
用户输入用户名密码，点击登录
    ↓
调用 /login 接口，后端返回 JWT Token
    ↓
前端存储 Token（localStorage）
    ↓
跳转到首页
    ↓
后续请求自动携带 Token（Axios 拦截器）
    ↓
Token 过期或无效 → 后端返回 NOT_LOGIN
    ↓
Axios 拦截器捕获 → 清除 Token → 跳转登录页
```

## 动手练习

### 练习 1：登录流程测试

1. 清空 localStorage，刷新页面，观察是否跳转到登录页
2. 登录成功后，观察 localStorage 中是否存储了 token
3. 点击退出登录，观察是否清除 token 并跳转

### 练习 2：Axios 拦截器处理 401

在 Axios 响应拦截器中，当收到 NOT_LOGIN 时：
1. 清除 localStorage 中的 token
2. 显示提示信息
3. 跳转到登录页

## 常见错误排查

| 错误现象 | 原因 | 解决方案 |
|---------|------|---------|
| 登录后仍跳回登录页 | Token 未存储或请求未携带 | 检查 localStorage 和 Axios 拦截器 |
| 刷新后登录状态丢失 | 未持久化存储 | 使用 localStorage 存储 token |
| 路由守卫不生效 | 守卫逻辑错误 | 检查 to.path 判断和 next() 调用 |
| 已登录还能访问登录页 | 缺少已登录跳转逻辑 | 在守卫中添加已登录判断 |

## 本节小结

登录认证是管理系统的第一道门槛。路由守卫负责拦截未授权访问，Axios 拦截器负责处理 Token 过期，两者配合实现了完整的认证流程。
