## 二、核心知识点

### 2.1 登录认证的基本概念

#### 是什么

登录认证是管理系统的第一道安全防线。用户输入用户名和密码，后端验证通过后返回一个"令牌"（Token），后续所有请求都携带这个 Token 来证明身份。

#### 为什么需要 Token

想象你去健身房：
- **没有 Token** = 每次去健身房都要在前台报姓名、身份证号、办卡的卡号，效率极低
- **有 Token** = 办一张会员卡，每次刷卡就能进，卡丢了补办一张即可

Token 就是这张"会员卡"，用户登录后获得，之后每次请求都"刷卡"（携带 Token）。

#### 认证流程图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Token 认证完整流程                            │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   第 1 步：用户登录                                                  │
│   ────────────────                                                   │
│   浏览器 ──▶ POST /login {username, password} ──▶ 后端              │
│                                                                     │
│   第 2 步：后端验证                                                  │
│   ────────────────                                                   │
│   后端验证用户名密码 ──▶ 生成 JWT Token ──▶ 返回给前端              │
│                                                                     │
│   第 3 步：前端存储 Token                                            │
│   ─────────────────────                                              │
│   前端收到 Token ──▶ 存入 localStorage ──▶ 跳转首页                 │
│                                                                     │
│   第 4 步：后续请求携带 Token                                        │
│   ─────────────────────────                                          │
│   浏览器 ──▶ GET /emps (headers: token=xxx) ──▶ 后端                │
│                                                                     │
│   第 5 步：Token 过期或无效                                          │
│   ────────────────────────                                           │
│   后端返回 401 / NOT_LOGIN ──▶ 前端清除 Token ──▶ 跳转登录页        │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

```vue
<!-- 文件名：src/views/login/index.vue -->

<template>
  <!-- 登录页面容器：全屏居中 -->
  <div class="login-container">
    <!-- el-card：卡片组件，带阴影效果 -->
    <el-card class="login-box">
      <!-- slot="header"：卡片头部区域 -->
      <div slot="header" class="login-title">Tlias 管理系统</div>

      <!-- el-form：表单组件 -->
      <!-- :model="form"：绑定表单数据 -->
      <!-- :rules="rules"：绑定校验规则 -->
      <!-- ref="loginForm"：给表单起名字，用于调用 validate 方法 -->
      <el-form :model="form" :rules="rules" ref="loginForm">

        <!-- 用户名输入框 -->
        <!-- prop="username"：对应 rules 中的校验规则 -->
        <el-form-item prop="username">
          <!-- v-model="form.username"：双向绑定 -->
          <!-- placeholder：输入框提示文字 -->
          <!-- prefix-icon：左侧图标 -->
          <el-input
            v-model="form.username"
            placeholder="用户名"
            prefix-icon="el-icon-user"
          ></el-input>
        </el-form-item>

        <!-- 密码输入框 -->
        <!-- type="password"：密码类型，输入时显示圆点 -->
        <el-form-item prop="password">
          <el-input
            v-model="form.password"
            type="password"
            placeholder="密码"
            prefix-icon="el-icon-lock"
            @keyup.enter.native="handleLogin"
          ></el-input>
        </el-form-item>

        <!-- 登录按钮 -->
        <el-form-item>
          <!-- type="primary"：蓝色主按钮 -->
          <!-- style="width: 100%;"：按钮宽度填满 -->
          <!-- @click="handleLogin"：点击调用登录方法 -->
          <!-- :loading="loading"：登录中时显示加载动画，禁用按钮 -->
          <el-button
            type="primary"
            style="width: 100%;"
            @click="handleLogin"
            :loading="loading"
          >
            登录
          </el-button>
        </el-form-item>
      </el-form>
    </el-card>
  </div>
</template>

<script>
export default {
  name: 'LoginPage',

  data() {
    return {
      // loading：控制登录按钮的加载状态
      // true = 显示加载动画，按钮不可点击
      // false = 正常状态
      loading: false,

      // form：表单数据
      form: {
        username: 'admin',    // 默认用户名（开发时方便测试）
        password: '123456'    // 默认密码
      },

      // rules：表单校验规则
      rules: {
        username: [
          // required: true 表示必填
          // message: 校验失败时的提示
          // trigger: 'blur' 表示失去焦点时触发
          { required: true, message: '请输入用户名', trigger: 'blur' }
        ],
        password: [
          { required: true, message: '请输入密码', trigger: 'blur' },
          // min: 6 表示最少 6 个字符
          { min: 6, message: '密码长度不能少于 6 位', trigger: 'blur' }
        ]
      }
    }
  },

  methods: {
    // handleLogin：处理登录
    handleLogin() {
      // 第 1 步：表单校验
      // this.$refs.loginForm.validate()：调用 ElementUI 的表单校验
      // valid：校验结果，true 表示通过
      this.$refs.loginForm.validate(valid => {
        if (valid) {
          // 第 2 步：设置加载状态
          // 防止用户重复点击，同时给用户视觉反馈
          this.loading = true

          // 第 3 步：发送登录请求
          // this.$request 是封装好的 axios
          this.$request.post('/login', this.form).then(token => {
            // 第 4 步：存储 Token
            // localStorage：浏览器本地存储，关闭页面后数据仍在
            // sessionStorage：关闭页面后数据清除
            localStorage.setItem('token', token)

            // 第 5 步：显示成功提示
            this.$message.success('登录成功')

            // 第 6 步：跳转到首页
            // this.$router.push('/')：编程式导航，跳转到首页
            this.$router.push('/')
          }).catch(() => {
            // 登录失败（用户名或密码错误）
            // 不需要额外处理，因为响应拦截器已经显示了错误提示
          }).finally(() => {
            // 第 7 步：无论成功失败，都关闭加载状态
            // finally 保证 loading 一定会被重置，避免按钮一直转圈
            this.loading = false
          })
        }
      })
    }
  }
}
</script>
```

### 2.4 路由守卫 —— 系统的"门禁系统"

#### 是什么

路由守卫（Navigation Guards）是 Vue Router 提供的钩子函数，在路由跳转前后执行，用于权限校验、日志记录等。

#### 为什么需要路由守卫

想象一栋办公楼：
- **没有路由守卫** = 大门敞开，任何人都能直接进入任何办公室
- **有路由守卫** = 门口有保安，检查门禁卡（Token），没卡的不让进

#### 全局前置守卫 beforeEach

```javascript
// src/router/index.js（Vue2 版本）

import Vue from 'vue'
import VueRouter from 'vue-router'

// 第 1 步：让 Vue 使用 VueRouter 插件
Vue.use(VueRouter)

// 第 2 步：定义路由配置
const routes = [
  {
    path: '/login',
    component: () => import('../views/login/index.vue'),
    meta: { title: '登录' }
  },
  {
    path: '/',
    component: () => import('../views/layout/index.vue'),
    redirect: '/dept',  // 重定向到部门管理
    children: [
      {
        path: 'dept',
        component: () => import('../views/dept/index.vue'),
        meta: { title: '部门管理' }
      },
      {
        path: 'emp',
        component: () => import('../views/emp/index.vue'),
        meta: { title: '员工管理' }
      }
    ]
  }
]

// 第 3 步：创建路由实例
const router = new VueRouter({
  mode: 'history',    // 路由模式：history 或 hash
  routes
})

// ==================== 全局前置守卫 ====================
// router.beforeEach：在每次路由跳转前执行
// to：目标路由对象（要跳到哪里去）
// from：来源路由对象（从哪来）
// next：放行函数，必须调用才能继续跳转

router.beforeEach((to, from, next) => {
  // 第 1 步：从 localStorage 获取 token
  const token = localStorage.getItem('token')

  // 第 2 步：判断目标路径是否是登录页
  if (to.path === '/login') {
    // 如果已经登录，跳转到首页（避免重复登录）
    if (token) {
      next('/')   // 已登录，去首页
    } else {
      next()      // 未登录，放行去登录页
    }
  } else {
    // 目标路径不是登录页，需要校验登录状态
    if (token) {
      next()      // 有 token，放行
    } else {
      next('/login')  // 没有 token，跳转到登录页
    }
  }
})

// 第 4 步：导出路由实例
export default router
```

#### 守卫执行流程图

```
用户点击链接或输入 URL
       │
       ▼
  router.beforeEach 触发
       │
       ▼
  ┌─────────────────────────┐
  │  to.path === '/login' ? │
  └─────────────────────────┘
       │
   是 ◀─┴─▶ 否
   │         │
   ▼         ▼
有token?   有token?
   │         │
 是├─▶next('/') 是├─▶next()
   │         │
 否├─▶next()   否├─▶next('/login')
```

#### 登录后跳转原页面（高级）

```javascript
// 改进版路由守卫：支持登录后跳回原来的页面

router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token')

  if (to.path === '/login') {
    if (token) {
      next('/')
    } else {
      next()
    }
  } else {
    if (token) {
      next()
    } else {
      // 将目标路径保存到地址栏的查询参数中
      // 登录成功后读取这个参数，跳回原页面
      next(`/login?redirect=${encodeURIComponent(to.fullPath)}`)
    }
  }
})

// 登录成功后读取 redirect 参数
// login.vue 中：
handleLogin() {
  this.$request.post('/login', this.form).then(token => {
    localStorage.setItem('token', token)
    this.$message.success('登录成功')

    // 获取 redirect 参数
    const redirect = this.$route.query.redirect
    // 如果有 redirect，跳转到原页面；否则跳转到首页
    this.$router.push(redirect || '/')
  })
}
```

### 2.5 hash 模式 vs history 模式

#### 是什么

Vue Router 支持两种路由模式，决定了 URL 的样子和浏览器刷新时的行为。

#### 两种模式对比

| 特性 | hash 模式 | history 模式 |
|------|-----------|--------------|
| URL 样子 | `http://localhost/#/dept` | `http://localhost/dept` |
| 实现原理 | 利用 URL 的 # 锚点，不会发送给服务器 | 利用 HTML5 History API |
| 浏览器刷新 | 正常（# 后的内容不发给服务器） | 需要服务器配置，否则 404 |
| 兼容性 | 所有浏览器都支持 | 需要 IE9+ |
| 美观度 | 有 #，不太美观 | 无 #，更像普通 URL |
| 生产环境配置 | 不需要额外配置 | 需要配置服务器回退到 index.html |

#### 模式选择建议

```
开发环境：两种都可以，history 更美观
生产环境：
  ├─ 如果你能配置服务器（Nginx/Apache）──▶ 选 history
  └─ 如果你不能配置服务器 ──────────────▶ 选 hash
```

#### 生产环境 history 模式的服务器配置

```nginx
# Nginx 配置
server {
  listen 80;
  server_name example.com;
  root /var/www/tlias-frontend/dist;
  index index.html;

  location / {
    # 关键配置：所有请求都回退到 index.html
    # 让 Vue Router 处理路由
    try_files $uri $uri/ /index.html;
  }
}
```

### 2.6 Axios 响应拦截器处理 Token 过期

#### 是什么

Token 通常有过期时间（如 2 小时）。过期后后端会返回特定错误码，前端需要捕获这个错误，清除 Token 并跳转登录页。

#### 完整拦截器代码

```javascript
// src/utils/request.js（Vue2 版本）

import axios from 'axios'
import { Message } from 'element-ui'
import router from '../router'

const request = axios.create({
  baseURL: '/api',
  timeout: 10000
})

// 请求拦截器：添加 Token
request.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.token = token
    }
    return config
  }
)

// 响应拦截器：处理错误和 Token 过期
request.interceptors.response.use(
  response => {
    const res = response.data

    if (res.code === 0) {
      Message.error(res.msg || '请求失败')

      // ========== Token 过期处理 ==========
      // 后端返回 NOT_LOGIN 表示 Token 无效或过期
      if (res.msg === 'NOT_LOGIN') {
        // 第 1 步：清除本地 Token
        localStorage.removeItem('token')

        // 第 2 步：提示用户
        Message.warning('登录已过期，请重新登录')

        // 第 3 步：跳转到登录页
        // 使用 replace 而不是 push，避免用户点返回回到已失效的页面
        router.replace('/login')
      }

      return Promise.reject(new Error(res.msg))
    }

    return res.data
  },
  error => {
    // HTTP 错误（如 404、500）
    if (error.response) {
      const status = error.response.status
      if (status === 401) {
        // HTTP 401 未授权
        localStorage.removeItem('token')
        Message.warning('登录已过期，请重新登录')
        router.replace('/login')
      } else if (status === 403) {
        Message.error('没有权限执行此操作')
      } else if (status === 404) {
        Message.error('请求的资源不存在')
      } else if (status >= 500) {
        Message.error('服务器错误，请稍后重试')
      }
    } else {
      Message.error(error.message || '网络错误')
    }
    return Promise.reject(error)
  }
)

export default request
```

## 四、常见错误排查

### 4.1 依赖配置问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| `router is not defined` | 拦截器中引入路由方式错误 | 确保 `import router from '../router'` 路径正确 |
| `Cannot read property 'push' of undefined` | Vue 原型上没挂载 $router | 确保 `new Vue({ router })` 传入了 router |
| 路由守卫不生效 | 守卫逻辑错误 | 检查 `to.path` 判断和 `next()` 调用 |
| 刷新后 404 | history 模式服务器未配置 | 配置 Nginx `try_files` 或改用 hash 模式 |

### 4.2 参数请求问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 登录后仍跳回登录页 | Token 未存储或请求未携带 | 检查 localStorage 和 Axios 拦截器 |
| 请求 401 未跳转 | 响应拦截器没处理 | 检查拦截器中是否处理了 401 / NOT_LOGIN |
| Token 存在但请求没携带 | 请求头字段名不匹配 | 确认后端读取的是 `token` 还是 `Authorization` |
| 登录接口返回了 Token 但存不进去 | localStorage 被禁用 | 检查浏览器隐私模式设置 |

### 4.3 代码逻辑问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 已登录还能访问登录页 | 缺少已登录跳转逻辑 | 在守卫中添加 `if (token && to.path === '/login') next('/')` |
| 刷新后登录状态丢失 | 未持久化存储 | 使用 localStorage（不是内存变量） |
| 退出后点返回还能进系统 | 用了 `push` 而不是 `replace` | 跳转登录页用 `router.replace()` |
| 路由守卫死循环 | next 调用逻辑错误 | 确保每个分支都调用了 next() 且不会循环 |
| 布局页面闪烁 | 未登录时先渲染了布局 | 确保守卫在渲染前执行（守卫总是在渲染前） |
| Vue3 中 `$route` 报错 | 组合式 API 中获取方式不同 | 使用 `useRoute()` 钩子 |

### 4.4 性能安全问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| Token 存在 XSS 风险 | localStorage 可被 JS 读取 | 配合后端设置 httpOnly Cookie（更安全的方案） |
| Token 长期有效 | 没有过期机制 | 后端设置 Token 过期时间，前端配合刷新机制 |
| 密码明文传输 | 没做加密 | 使用 HTTPS，前端可做简单 MD5 加密 |
| 登录接口被暴力破解 | 没有限流 | 后端做登录失败次数限制和验证码 |
| 敏感信息存在 localStorage | 容易被读取 | Token 可以存，用户密码等敏感信息不要存 |
| 路由守卫被绕过 | 只做了前端校验 | 后端每个接口都要独立校验 Token |

**核心要点回顾**：

1. 登录页面用 `el-form` + `el-input` + 表单校验实现，登录时显示 `loading` 防止重复提交

3. 布局组件用 `el-container` + `el-aside` + `el-menu` 实现侧边栏导航，`router-view` 显示主内容
4. 路由守卫 `router.beforeEach` 在每次跳转前检查 Token，无 Token 跳登录页
5. hash 模式兼容性好但有 `#`，history 模式美观但需要服务器配置
6. Axios 响应拦截器捕获 `NOT_LOGIN`，自动清除 Token 并跳转登录页
7. 退出登录时清除 Token 并跳转，用 `replace` 避免返回历史记录
