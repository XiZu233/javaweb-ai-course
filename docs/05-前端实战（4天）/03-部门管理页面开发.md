# 部门管理页面开发

## 学习目标

- 掌握 ElementUI/ElementPlus 表格和表单的综合使用
- 理解对话框（Dialog）的打开/关闭逻辑
- 能够实现完整的增删改查页面

## 核心知识点

### 1. 页面结构分析

部门管理页面包含以下元素：
- **操作区**：新增按钮
- **数据区**：表格展示部门列表
- **对话框**：新增/编辑的表单

### 2. 版本 A 完整代码（Vue2 + ElementUI）

```vue
<template>
  <div>
    <el-button type="primary" icon="el-icon-plus" @click="handleAdd">新增部门</el-button>

    <el-table :data="deptList" style="margin-top: 20px;" border>
      <el-table-column type="index" label="序号" width="80" align="center"></el-table-column>
      <el-table-column prop="name" label="部门名称"></el-table-column>
      <el-table-column prop="createTime" label="创建时间"></el-table-column>
      <el-table-column prop="updateTime" label="更新时间"></el-table-column>
      <el-table-column label="操作" width="180" align="center">
        <template slot-scope="scope">
          <el-button type="text" @click="handleEdit(scope.row)">编辑</el-button>
          <el-button type="text" style="color: #f56c6c;" @click="handleDelete(scope.row.id)">删除</el-button>
        </template>
      </el-table-column>
    </el-table>

    <!-- 新增/编辑对话框 -->
    <el-dialog :title="dialogTitle" :visible.sync="dialogVisible" width="400px">
      <el-form :model="form" label-width="80px">
        <el-form-item label="部门名称">
          <el-input v-model="form.name"></el-input>
        </el-form-item>
      </el-form>
      <span slot="footer">
        <el-button @click="dialogVisible = false">取消</el-button>
        <el-button type="primary" @click="submit">确定</el-button>
      </span>
    </el-dialog>
  </div>
</template>

<script>
export default {
  data() {
    return {
      deptList: [],
      dialogVisible: false,
      dialogTitle: '新增部门',
      form: { id: null, name: '' }
    }
  },
  created() {
    this.loadData()
  },
  methods: {
    loadData() {
      this.$request.get('/depts').then(data => {
        this.deptList = data
      })
    },
    handleAdd() {
      this.form = { id: null, name: '' }
      this.dialogTitle = '新增部门'
      this.dialogVisible = true
    },
    handleEdit(row) {
      this.form = { ...row }  // 浅拷贝，避免直接修改表格数据
      this.dialogTitle = '编辑部门'
      this.dialogVisible = true
    },
    submit() {
      const api = this.form.id
        ? this.$request.put('/depts', this.form)
        : this.$request.post('/depts', this.form)
      api.then(() => {
        this.$message.success(this.form.id ? '修改成功' : '新增成功')
        this.dialogVisible = false
        this.loadData()
      })
    },
    handleDelete(id) {
      this.$confirm('确认删除该部门？', '提示', { type: 'warning' }).then(() => {
        this.$request.delete(`/depts/${id}`).then(() => {
          this.$message.success('删除成功')
          this.loadData()
        })
      })
    }
  }
}
</script>
```

### 3. 关键逻辑解析

**新增 vs 编辑复用对话框**：
- 通过 `form.id` 判断是新增还是编辑
- 有 id 调用 PUT 接口，无 id 调用 POST 接口
- 打开对话框前重置表单数据

**数据拷贝**：
```javascript
this.form = { ...row }  // ES6 展开运算符浅拷贝
```
直接赋值 `this.form = row` 会导致修改表单时表格数据同步变化。

### 4. 版本 B 差异（Vue3 + ElementPlus）

```vue
<script setup lang="ts">
import { ref, onMounted } from 'vue'
import { ElMessage, ElMessageBox } from 'element-plus'
import request from '@/utils/request'

const deptList = ref([])
const dialogVisible = ref(false)
const dialogTitle = ref('新增部门')
const form = ref({ id: null as number | null, name: '' })

const loadData = async () => {
  const res = await request.get('/depts')
  deptList.value = res
}

const handleAdd = () => {
  form.value = { id: null, name: '' }
  dialogTitle.value = '新增部门'
  dialogVisible.value = true
}

const handleEdit = (row: any) => {
  form.value = { ...row }
  dialogTitle.value = '编辑部门'
  dialogVisible.value = true
}

const submit = async () => {
  if (form.value.id) {
    await request.put('/depts', form.value)
  } else {
    await request.post('/depts', form.value)
  }
  ElMessage.success(form.value.id ? '修改成功' : '新增成功')
  dialogVisible.value = false
  loadData()
}

const handleDelete = async (id: number) => {
  await ElMessageBox.confirm('确认删除该部门？', '提示', { type: 'warning' })
  await request.delete(`/depts/${id}`)
  ElMessage.success('删除成功')
  loadData()
}

onMounted(loadData)
</script>
```

## 常见错误排查

| 错误现象 | 原因 | 解决方案 |
|---------|------|---------|
| 编辑后表格数据同步变化 | 直接引用对象 | 使用 { ...row } 浅拷贝 |
| 对话框关闭后数据残留 | 未重置表单 | 打开对话框前重置 form |
| 新增成功列表未刷新 | 未调用 loadData | submit 成功后调用 loadData |

## 本节小结

部门管理页面是管理后台的基础模板：表格展示数据 + 对话框编辑数据。掌握这个模式，你就能快速开发大多数 CRUD 页面。
