### 2.1 页面结构分析 —— 比部门管理更复杂的场景

#### 是什么

### 2.2 多条件搜索与日期范围选择

#### 是什么

多条件搜索允许用户同时按多个字段筛选数据。日期范围选择用于筛选某个时间段内的记录。

#### 搜索表单实现

```vue
<!-- ========== 搜索表单 ========== -->
<!-- :inline="true"：行内表单，表单项横向排列 -->
<!-- :model="searchForm"：绑定搜索条件数据对象 -->
<el-form :inline="true" :model="searchForm" style="margin-bottom: 20px;">

  <!-- 姓名搜索 -->
  <!-- clearable：显示清空按钮，点击可清空输入 -->
  <el-form-item label="姓名">
    <el-input
      v-model="searchForm.name"
      placeholder="请输入姓名"
      clearable>
    </el-input>
  </el-form-item>

  <!-- 性别下拉选择 -->
  <!-- clearable：允许清空选择 -->
  <el-form-item label="性别">
    <el-select v-model="searchForm.gender" placeholder="请选择" clearable>
      <!-- :value="1"：选中时 searchForm.gender = 1 -->
      <el-option label="男" :value="1"></el-option>
      <el-option label="女" :value="2"></el-option>
    </el-select>
  </el-form-item>

  <!-- 入职日期范围 -->
  <!-- type="daterange"：日期范围选择器 -->
  <!-- range-separator="至"：两个日期之间的分隔文字 -->
  <!-- value-format="yyyy-MM-dd"：绑定值的格式（Vue2）-->
  <!-- value-format="YYYY-MM-DD"：绑定值的格式（Vue3）-->
  <el-form-item label="入职日期">
    <el-date-picker
      v-model="dateRange"
      type="daterange"
      range-separator="至"
      start-placeholder="开始日期"
      end-placeholder="结束日期"
      value-format="yyyy-MM-dd">
    </el-date-picker>
  </el-form-item>

  <!-- 操作按钮 -->
  <el-form-item>
    <!-- @click="loadData"：点击查询，重新加载数据 -->
    <el-button type="primary" icon="el-icon-search" @click="handleSearch">
      查询
    </el-button>
    <!-- @click="reset"：点击重置，清空搜索条件 -->
    <el-button icon="el-icon-refresh" @click="reset">
      重置
    </el-button>
  </el-form-item>
</el-form>
```

#### 搜索逻辑详解

```javascript
// ========== 数据定义 ==========
data() {
  return {
    // 搜索条件对象
    searchForm: {
      name: '',        // 姓名（模糊搜索）
      gender: null     // 性别（1=男，2=女，null=不限）
    },
    // 日期范围（el-date-picker 绑定的是数组）
    // 选择后格式：['2024-01-01', '2024-06-30']
    dateRange: [],
    // 分页参数
    page: 1,           // 当前页码
    pageSize: 10,      // 每页条数
    total: 0           // 总条数
  }
},

methods: {
  // loadData：加载员工列表（带搜索条件和分页）
  loadData() {
    // 构建请求参数
    // ...this.searchForm：将 searchForm 的所有属性展开到 params 中
    const params = {
      page: this.page,
      pageSize: this.pageSize,
      ...this.searchForm   // ES6 展开运算符，等同于 name: this.searchForm.name, gender: this.searchForm.gender
    }

    // 如果有日期范围，添加 begin 和 end 参数
    if (this.dateRange && this.dateRange.length === 2) {
      params.begin = this.dateRange[0]   // 开始日期
      params.end = this.dateRange[1]     // 结束日期
    }

    // 发送请求
    this.$request.get('/emps', { params }).then(data => {
      // 假设后端返回格式：{ list: [...], total: 100 }
      this.empList = data.list
      this.total = data.total
    })
  },

  // handleSearch：点击查询按钮
  handleSearch() {
    // 搜索时重置到第一页
    // 原因：用户可能在第 3 页，改了搜索条件后应该看第 1 页的结果
    this.page = 1
    this.loadData()
  },

  // reset：点击重置按钮
  reset() {
    // 清空搜索条件
    this.searchForm = { name: '', gender: null }
    this.dateRange = []
    // 重置到第一页
    this.page = 1
    // 重新加载数据
    this.loadData()
  }
}
```

#### ES6 展开运算符详解

```javascript
// 展开运算符 ... 的作用：将对象或数组"展开"

// 对象展开
const searchForm = { name: '张三', gender: 1 }
const params = { page: 1, ...searchForm }
// 结果：params = { page: 1, name: '张三', gender: 1 }

// 等价于：
const params2 = {
  page: 1,
  name: searchForm.name,
  gender: searchForm.gender
}

// 数组展开
const arr1 = [1, 2]
const arr2 = [...arr1, 3, 4]
// 结果：arr2 = [1, 2, 3, 4]
```

### 2.3 分页组件与后端分页联动

#### 是什么

分页用于将大量数据分成多页显示，避免一次性加载过多数据导致页面卡顿。

#### 前端分页 vs 后端分页

| 方式 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 前端分页 | 一次性加载所有数据，前端计算显示哪一页 | 切换页码快 | 首次加载慢，数据量大时内存占用高 | 数据量少（< 1000 条） |
| 后端分页 | 每次只请求当前页的数据 | 首次加载快，支持大数据量 | 切换页码需要发请求 | 数据量大（推荐） |

本项目使用**后端分页**。

#### 分页组件详解

```vue
<!-- ElementUI 分页组件 -->
<el-pagination
  <!-- 样式：右对齐，上边距 20px -->
  style="margin-top: 20px; text-align: right;"

  <!-- @current-change：页码变化时触发的事件 -->
  <!-- 用户点击页码或上一页/下一页时调用 -->
  @current-change="loadData"

  <!-- :current-page：当前页码（双向绑定） -->
  :current-page="page"

  <!-- :page-size：每页显示的条数 -->
  :page-size="pageSize"

  <!-- :total：总条数 -->
  :total="total"

  <!-- layout：控制分页组件显示哪些元素 -->
  <!-- total=总条数, prev=上一页, pager=页码, next=下一页 -->
  layout="total, prev, pager, next"
></el-pagination>
```

#### 分页联动流程

```
用户点击第 3 页
       │
       ▼
  @current-change 触发
       │
       ▼
  page 变为 3
       │
       ▼
  调用 loadData()
       │
       ▼
  发送请求：GET /emps?page=3&pageSize=10
       │
       ▼
  后端返回第 3 页的数据
       │
       ▼
  更新 empList 和 total
       │
       ▼
  表格显示第 3 页数据
```

### 2.4 枚举值映射（数字转文字）

#### 是什么

后端存储的性别、职位等字段通常用数字表示（如 1=男，2=女），但页面上需要显示中文。这个"数字转文字"的过程就是枚举值映射。

#### 实现方式对比

```javascript
// 方式 1：对象映射（推荐，简单直观）
// 在 data 中定义映射对象
data() {
  return {
    // 性别映射
    genderMap: { 1: '男', 2: '女' },
    // 职位映射
    jobMap: {
      1: '班主任',
      2: '讲师',
      3: '学工主管',
      4: '教研主管',
      5: '咨询师',
      6: '其他'
    }
  }
}

// 模板中使用
// {{ genderMap[scope.row.gender] }}  // 显示"男"或"女"
// {{ jobMap[scope.row.job] }}        // 显示职位名称

// 方式 2：计算属性（适合需要复杂逻辑的场景）
computed: {
  // 根据 gender 值返回对应的标签类型（ElementUI 的 type 属性）
  genderType() {
    return (gender) => {
      return gender === 1 ? 'primary' : 'danger'
    }
  }
}

// 方式 3：过滤器（Vue2 特有，Vue3 已移除）
// 在模板中使用：{{ scope.row.gender | genderFilter }}
filters: {
  genderFilter(value) {
    const map = { 1: '男', 2: '女' }
    return map[value] || '未知'
  }
}

// 方式 4：方法（Vue3 推荐替代过滤器）
// 在模板中使用：{{ formatGender(scope.row.gender) }}
methods: {
  formatGender(gender) {
    const map = { 1: '男', 2: '女' }
    return map[gender] || '未知'
  }
}
```

### 2.5 文件上传组件（头像上传）

#### 是什么

`el-upload` 是 ElementUI 提供的文件上传组件，支持点击上传、拖拽上传、图片预览等功能。

#### 头像上传实现

```vue
<!-- 在员工表单中使用头像上传 -->
<el-form-item label="头像">
  <!-- action：上传接口地址 -->
  <!-- :show-file-list="false"：不显示文件列表（头像只传一张） -->
  <!-- :on-success：上传成功后的回调 -->
  <!-- :before-upload：上传前的校验 -->
  <!-- :headers：自定义请求头（如 token） -->
  <el-upload
    class="avatar-uploader"
    action="/api/upload"
    :show-file-list="false"
    :on-success="handleAvatarSuccess"
    :before-upload="beforeAvatarUpload"
    :headers="uploadHeaders">

    <!-- 有头像时显示图片 -->
    <img v-if="form.image" :src="form.image" class="avatar">

    <!-- 没有头像时显示上传图标 -->
    <i v-else class="el-icon-plus avatar-uploader-icon"></i>
  </el-upload>
</el-form-item>

<script>
export default {
  data() {
    return {
      form: {
        image: ''   // 头像 URL
      },
      // 上传请求头（携带 token）
      uploadHeaders: {
        token: localStorage.getItem('token')
      }
    }
  },
  methods: {
    // handleAvatarSuccess：上传成功回调
    // res：后端返回的数据
    // file：上传的文件信息
    handleAvatarSuccess(res, file) {
      // 假设后端返回格式：{ code: 1, data: 'http://xxx.jpg' }
      if (res.code === 1) {
        // 将返回的图片 URL 保存到表单
        this.form.image = res.data
        this.$message.success('上传成功')
      } else {
        this.$message.error(res.msg || '上传失败')
      }
    },

    // beforeAvatarUpload：上传前校验
    // file：要上传的文件
    // 返回 true 允许上传，返回 false 阻止上传
    beforeAvatarUpload(file) {
      // 第 1 步：校验文件类型
      // file.type 是 MIME 类型，如 image/jpeg
      const isJPG = file.type === 'image/jpeg'
      const isPNG = file.type === 'image/png'
      const isImage = isJPG || isPNG

      // 第 2 步：校验文件大小（2MB = 2 * 1024 * 1024 字节）
      const isLt2M = file.size / 1024 / 1024 < 2

      // 第 3 步：如果不满足条件，提示错误
      if (!isImage) {
        this.$message.error('只支持 JPG 和 PNG 格式的图片!')
      }
      if (!isLt2M) {
        this.$message.error('图片大小不能超过 2MB!')
      }

      // 第 4 步：返回校验结果
      return isImage && isLt2M
    }
  }
}
</script>
```

### 2.6 表单校验（rules）

#### 是什么

表单校验用于在用户提交前检查输入是否合法，避免将错误数据发送到后端。

#### 校验规则详解

```javascript
// 表单校验规则
rules: {
  // 姓名校验
  name: [
    // 规则 1：必填
    { required: true, message: '请输入姓名', trigger: 'blur' },
    // 规则 2：长度限制
    { min: 2, max: 10, message: '长度在 2 到 10 个字符', trigger: 'blur' }
  ],

  // 性别校验
  gender: [
    { required: true, message: '请选择性别', trigger: 'change' }
  ],

  // 入职日期校验
  entrydate: [
    { required: true, message: '请选择入职日期', trigger: 'change' }
  ],

  // 所属部门校验
  deptId: [
    { required: true, message: '请选择所属部门', trigger: 'change' }
  ],

  // 手机号校验（自定义规则）
  phone: [
    { required: true, message: '请输入手机号', trigger: 'blur' },
    {
      // pattern：正则表达式校验
      pattern: /^1[3-9]\d{9}$/,
      message: '手机号格式不正确',
      trigger: 'blur'
    }
  ],

  // 邮箱校验（自定义规则）
  email: [
    { required: true, message: '请输入邮箱', trigger: 'blur' },
    {
      pattern: /^[\w-]+(\.[\w-]+)*@[\w-]+(\.[\w-]+)+$/,
      message: '邮箱格式不正确',
      trigger: 'blur'
    }
  ],

  // 自定义校验函数（最灵活）
  age: [
    { required: true, message: '请输入年龄', trigger: 'blur' },
    {
      // validator：自定义校验函数
      // rule：当前规则对象
      // value：当前字段的值
      // callback：校验完成后的回调，必须调用
      validator: (rule, value, callback) => {
        if (!Number.isInteger(value)) {
          // 调用 callback(new Error('错误信息')) 表示校验失败
          callback(new Error('年龄必须是整数'))
        } else if (value < 18 || value > 60) {
          callback(new Error('年龄必须在 18 到 60 之间'))
        } else {
          // 调用 callback() 表示校验通过
          callback()
        }
      },
      trigger: 'blur'
    }
  ]
}
```

#### 校验触发时机

| trigger 值 | 触发时机 | 适用场景 |
|-----------|----------|----------|
| `blur` | 输入框失去焦点时 | 文本输入框 |
| `change` | 选择器的值变化时 | 下拉选择、日期选择 |
| `submit` | 表单提交时 | 所有字段 |

### 2.7 表格多选与批量删除

#### 是什么

表格多选允许用户同时选择多条记录，然后执行批量操作（如批量删除）。

#### 实现步骤

```vue
<template>
  <div>
    <!-- 批量删除按钮 -->
    <!-- :disabled="selectedIds.length === 0"：没有选择时禁用按钮 -->
    <el-button
      type="danger"
      icon="el-icon-delete"
      @click="handleBatchDelete"
      :disabled="selectedIds.length === 0">
      批量删除
    </el-button>

    <!-- 表格 -->
    <!-- @selection-change：多选状态变化时触发 -->
    <el-table
      :data="empList"
      border
      @selection-change="handleSelectionChange">

      <!-- type="selection"：开启多选列（表格最左侧的复选框） -->
      <el-table-column type="selection" width="55" align="center">
      </el-table-column>

      <!-- 其他列... -->
      <el-table-column prop="name" label="姓名"></el-table-column>

      <!-- 操作列 -->
      <el-table-column label="操作" width="180" align="center">
        <template slot-scope="scope">
          <!-- 单行删除传入数组（兼容批量删除接口） -->
          <el-button type="text" @click="handleDelete([scope.row.id])">
            删除
          </el-button>
        </template>
      </el-table-column>
    </el-table>
  </div>
</template>

<script>
export default {
  data() {
    return {
      empList: [],
      // selectedIds：存储选中的员工 ID 数组
      // 例如：[1, 3, 5] 表示选中了 ID 为 1、3、5 的员工
      selectedIds: []
    }
  },
  methods: {
    // handleSelectionChange：多选状态变化时触发
    // rows：当前选中的所有行数据数组
    handleSelectionChange(rows) {
      // 从选中的行中提取 ID，组成数组
      // map 方法：对数组每个元素执行函数，返回新数组
      this.selectedIds = rows.map(row => row.id)
    },

    // handleDelete：删除（支持单条和批量）
    // ids：要删除的 ID 数组，如 [1] 或 [1, 2, 3]
    handleDelete(ids) {
      // 确认对话框
      this.$confirm(`确认删除选中的 ${ids.length} 条记录？`, '提示', {
        type: 'warning'
      }).then(() => {
        // 将 ID 数组用逗号连接成字符串
        // [1, 2, 3] → "1,2,3"
        this.$request.delete(`/emps/${ids.join(',')}`).then(() => {
          this.$message.success('删除成功')
          // 清空选择
          this.selectedIds = []
          // 刷新列表
          this.loadData()
        })
      })
    },

    // handleBatchDelete：批量删除
    handleBatchDelete() {
      // 直接调用 handleDelete，传入选中的 ID 数组
      this.handleDelete(this.selectedIds)
    }
  }
}
```

### 练习 3：表单校验增强

**目标**：给员工表单添加更严格的校验。

**要求**：

1. 姓名：必填，2-10 个字符，只能中文或英文
2. 入职日期：不能选择未来日期
3. 头像：上传时校验尺寸（建议 200x200 像素）

**提示**：

```javascript
// 姓名正则：中文或英文
{ pattern: /^[一-龥a-zA-Z]+$/, message: '只能输入中文或英文', trigger: 'blur' }

// 日期校验：不能选未来
{
  validator: (rule, value, callback) => {
    if (value && new Date(value) > new Date()) {
      callback(new Error('入职日期不能是未来日期'))
    } else {
      callback()
    }
  },
  trigger: 'change'
}
```

## 四、常见错误排查

### 4.1 依赖配置问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| `el-date-picker` 不显示 | 未引入日期组件（按需加载时） | `Vue.use(DatePicker)` 或完整引入 |
| `el-upload` 上传失败 | action 地址错误或后端未处理 | 检查 action URL 和后端接口 |
| 头像上传后无法预览 | 后端返回的 URL 不正确 | 确认后端返回的是可访问的完整 URL |
| `el-select` 选项不显示 | `v-for` 的 `:key` 重复 | 确保每个 option 的 key 唯一 |

### 4.2 参数请求问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 分页页码不跳转 | `@current-change` 未绑定 | 确保事件绑定正确 |
| 搜索后未回到第一页 | 未重置 page | 搜索时设置 `this.page = 1` |
| 日期范围搜索无效 | 参数名不匹配 | 确认后端接收的是 `begin`/`end` 还是其他 |
| 批量删除只删了一条 | ID 拼接格式不对 | 检查 `ids.join(',')` 的输出 |
| 头像上传没带 token | 未设置 headers | 给 `el-upload` 添加 `:headers` |

### 4.3 代码逻辑问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 多选后批量删除按钮未启用 | `selectedIds` 未更新 | 检查 `@selection-change` 绑定 |
| 编辑时职位下拉不显示选中 | `job` 类型不匹配 | 确保 `job` 是数字不是字符串 |
| 表单提交后校验错误还在 | 未重置校验状态 | 调用 `this.$refs.formRef.clearValidate()` |
| 重置搜索后日期还在 | `dateRange` 未清空 | 重置时设置 `this.dateRange = []` |
| 枚举值显示 undefined | map 中找不到对应 key | 添加默认值：`jobMap[row.job] || '未知'` |
| 部门下拉框为空 | 部门列表未加载 | 在 created 中调用 `loadDeptList()` |
| Vue3 中 `v-for` 报错 | 没有 `:key` | 给 `v-for` 添加 `:key` |

### 4.4 性能安全问题

| 错误现象 | 原因分析 | 解决方案 |
|----------|----------|----------|
| 表格数据量大时卡顿 | 没有分页，一次性渲染 | 使用后端分页，每页 10-20 条 |
| 头像图片太大加载慢 | 没限制上传图片尺寸 | 在 `before-upload` 中限制尺寸 |
| 批量删除误操作 | 没有二次确认 | 使用 `this.$confirm` / `ElMessageBox.confirm` |
| 搜索条件 XSS 风险 | 用户输入直接显示 | ElementUI 会自动转义，后端也要校验 |
| 文件上传类型未限制 | 可能上传恶意文件 | 在 `before-upload` 中严格校验 type |
| 分页参数被篡改 | 用户手动修改 URL | 后端校验 page 和 pageSize 的合法性 |

**核心要点回顾**：

1. 多条件搜索用 `el-form:inline` 横向排列，`...searchForm` 展开搜索条件
2. 搜索时记得重置 `page = 1`，否则用户可能在第 3 页搜索，看不到结果
3. 分页组件 `@current-change` 绑定加载方法，`layout` 控制显示元素
4. 枚举值映射用对象（`jobMap`）最简洁，模板中用 `{{ map[key] }}`
5. 头像上传用 `el-upload`，注意配置 `action`、`on-success`、`before-upload`
6. 表单校验支持 `required`、`pattern`、`validator` 等多种规则
7. 批量删除用 `type="selection"` 开启多选，`@selection-change` 获取选中项
